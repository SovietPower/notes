### 概念



- FLOPS (floating point operations per second)：每秒浮点运算次数，表示计算速度。是一个衡量硬件性能的指标。
- FLOPs (floating point operations)：浮点运算数，表示计算量。可用来衡量算法/模型的复杂度。

**算子**

> 算子是 AI 框架中最基础的概念，它是对张量 (Tensor) 执行基本操作的集合，包括四则运算、数学函数、甚至是对张量元数据的修改，如维度压缩（Squeeze），维度修改（reshape）等。
> Kernel 是算子的底层实现，通常是用 C++ 等语言编写的。Kernel 通常不感知微分的概念，也不支持自动梯度计算。
> 中间态是指在计算过程中产生的临时数据，它可以是张量，也可以是其他类型的数据。
>
> 简单来说：
>
> - 算子：从数学角度定义的操作。
> - Kernel：算子的底层实现。
> - 中间态：计算过程中的临时数据。
>
> 深度学习模型由一个个计算单元组成，称这些计算单元为算子 (Operator，简称 OP)。在网络模型中，算子对应层中的计算逻辑，例如：卷积层 (Convolution Layer) 是一个算子；全连接层 (Fully-connected Layer, FC layer) 中的权值求和过程，是一个算子。
>
> 实现上，算子可以是一个类或函数（编程中的函数，不是数学意义的函数）。

> 函数是从数到数的映射，泛函是从函数到数的映射，算子是从函数到函数的映射。
> 有时这几个词可以混用，比如可以把数当作常函数，那么普通的函数也可以看作泛函或算子；再比如考虑从算子到算子的映射，你仍然可以叫它算子。
>
> 可以认为算子是把一个函数变成另一个函数的东西。
> 梯度（求导）就是一个算子，将一个函数映射到另一个函数。

**张量**

张量就是任意维度的数组，是对向量和矩阵的扩展。

张量的形状 (shape) 即每个维度的大小。比如：shape = (4, 20, 20, 3) 可以代表有4张图，每张图的宽和高都是20（400个像素），每个像素点都由 RGB 3色组成。

> 在深度学习领域，多维数据通过多维数组存储，比如：卷积神经网络的特征图 (Feature Map) 通常用四维数组保存，即 4D 格式：
>
> N：Batch数量，例如图像的数量。
> H：Height，特征图高度，即垂直高度方向的像素个数。
> W：Width，特征图宽度，即水平宽度方向的像素个数。
> C：Channels，特征图通道，例如彩色RGB图像的Channels为3。
>
> 由于数据只能线性存储，因此这四个维度有对应的顺序。不同深度学习框架会按照不同的顺序存储特征图数据，比如 Caffe 中排列顺序为 [Batch, Channels, Height, Width]，即 NCHW；TensorFlow 中，排列顺序为 [Batch, Height, Width, Channels]，即 NHWC。
>
> 以一张格式为 RGB 的图片为例，NCHW 中，C 排列在外层，实际存储的是“RRRRRRGGGGGGBBBBBB”，即同一通道的所有像素值顺序存储在一起；而 NHWC 中 C 排列在最内层，实际存储的则是“RGBRGBRGBRGBRGBRGB”，即不同通道的同一位置的像素值顺序存储在一起。

### 梯度下降

基本思想为：为了尽快找到函数的极值，选择往函数变化最快的方向前进；如果要找误差函数的极小值，那么应该朝函数的负梯度方向搜索，即往负梯度方向更新参数。
函数变化最快的方向就是函数的梯度/导数。

原理：泰勒公式：如果函数$f(x)$在含$x_0$的某个开区间$(a,b)$内有 n+1 阶导数，则对$\forall x\in(a,b)$，$f(x)=f(x_0)+\sum_{i=1}^n\frac{f^{(i)}(x_0)}{i!}(x-x_0)^i+R_n(x)$，其中余项$R_n(x)$是误差。

梯度下降实际上是对误差函数$f$的泰勒级数进行一阶展开，得到近似函数：$f(x)=f(x_0)+f'(x_0)(x-x_0)$，其中目前的模型为$x_0$。
然后我们希望找到新的模型参数$x$使得$f(x)$尽量小。
设新的参数$x=x_0+\lambda \vec{d}$，所以$f(x)=f(x_0)+\lambda \vec d\cdot f'(x_0)$，通过[这里的推导](https://zhuanlan.zhihu.com/p/152566066)可以得出，当$x=x_0-\lambda f'(x_0)$时，新模型的（拟合后的）误差最小。其中$f'(x_0)$就是梯度，$\lambda$是未知量，用学习率超参数表示。

Gradient Descent：$\theta^{i+1}=\theta^{i}-\eta\nabla L(\theta^i)$
$\eta$为步长或学习率，当步长小时，收敛慢；步长大时，容易越过极小值点，导致错误或收敛更慢。
因此通常随着迭代次数增加，减小步长。

Vanilla Gradient Descent：$$\theta^{i+1}=\theta^{i}-\frac{\eta}{\sqrt{i+1}}\nabla L(\theta^i)$$
随迭代次数的增加，减少学习率。

梯度下降有三种形式：

- **Batch Gradient Descend（批量梯度下降）**

每次训练，使用所有样本计算误差，然后更新参数。
可以直接收敛到最优解。
但每次迭代训练很慢；空间开销极大；只向误差函数梯度方向前进，不具备跳出局部最优解的能力。

- **Stochastic Gradient Descent（随机梯度下降）**

每次训练，使用单个样本计算误差，然后更新参数。
每次迭代更快；增加了随机性（仅通过单个样本计算误差，可能不准确，与最终的模型不符），有一定跳出局部最优解的能力。

收敛更慢：虽然单次迭代不一定向最优方向前进，但整体上（多次迭代后）还是正确的，但收敛的路线会曲折（噪声多）。
SGD 在离最优点比较远时，收敛得比较快；而在接近最优点时，比较难收敛到最优点。

- **Mini-Batch Stochastic Gradient Descent（小批量随机梯度下降）**

常用的方法。
每次训练，使用部分样本计算误差。
比较精确，同时也比较快；多样本的计算能够并行，利用硬件。

三种形式就是不同 batch size 的梯度下降，需要根据实际情况调整该值。

> 使用所有数据完成一次训练，称为一个 epoch。
> 因此 batch size 越小，进行一个 epoch 所需的迭代次数就越多，所花的时间就越多（需要更多次的梯度下降）。
>
> 增大 batch size 能够充分利用硬件并行计算的能力；进行一个 epoch 所需的时间少。
> 但当 batch size 过大时，内存不能容纳所有样本（或者 GPU 显存？），就无法进行高效计算（需要从硬盘或网络等其它介质读取）；每个 epoch 中的迭代次数少，因此想达到相同的精度，需要的 epoch 就更多。
>
> 因此，batch size 越大，进行一个 epoch 所需的时间越少，但需要的 epoch 越多。
> 所以 batch size 不能过大也不能过小。降低和提高 batch size 都有可能降低训练效果。
>
> 这与==数据的多样性==有关。
> 一轮迭代中，使用的数据的多样性越好、差异性越明显，效果越好。
> 如果使用的数据都差不多，那么用1个样本和用1000个样本计算出来的梯度是差不多的（梯度会除以样本数量），对模型的贡献差不多，后者多处理的样本就白算了，并不会提高训练效果。
> 因此如果有足够大的样本（容易取到多样的样本），或样本的质量好、差异性好，那么可以使用更大的 batch size；否则 batch size 就不需要多大。
> 简单来说，如果数据有 n 个类，那么 batch size 差不多取 10n，再大就很难了。
> （此外还取决于优化算法）

**P5 Where does the error come from?**

- bias：$f^*$的平均值距离实际$\widehat{f}$的偏差。
- variance：各预测函数 $f^*$ 距离它们平均值的方差。
- 越简单的 model 受样本数据的影响越小，得到的预测函数$f^*$的方差(variance)越小，但其平均值$\overline{f}$距离正确预测的偏差(bias)越大。
- 如果误差主要来自偏差，则 model 过简单(underfitting)；如果主要来自方差，则 model 过复杂(overfitting)。
- 如果模型与训练数据误差大，则偏差太大(underfitting)；如果模型与训练数据误差小，但与测试数据误差大，则方差太大(overfitting)。
- 偏差太大：增加特征；设计更复杂的模型。
- 方差太大：增加训练数据量；正则化(regularization)。
- N-fold Cross Validation

**P6 Gradient Descent**

- 梯度下降：$\theta^{i+1}=\theta^{i}-\eta\nabla L(\theta^i)$

- **Vanilla Gradient Descent**

随迭代次数的增加，减少 learning rate。
$$\theta^{i+1}=\theta^{i}-\frac{\eta}{\sqrt{i+1}}\nabla L(\theta^i)$$

- **Adagard Gradient Descent**

对每个参数$w$使用不同的 learning rate，且根据历史微分结果调整。
令学习率为 $\eta^i=\frac{\eta}{\sqrt{i+1}}$，对参数$w$的偏微分为 $g^i=\frac{\partial C(\theta^i)}{\partial w}$，过去所有偏微分的均方根(root mean square)为 $\sigma^i={\sqrt{\frac{1}{i+1}\sum_{i=0}^i(g^i)^2}}$。
$$w^{i+1}=w^i-\frac{\eta^i}{\sigma^i}g^i =w^i-\frac{\eta}{\sqrt{\sum_{i=0}^t(g^i)^2}}g^i$$

但这似乎与直觉不符：偏微分$g$越大，似乎代表离极值点越远，所以系数应该更大；但除数以$g$为分母，$g$越大系数会越小。
事实上，上述直觉仅在函数有一个参数时成立。当函数有多个参数时，$\frac{一阶微分}{二阶微分}$的大小才能一定反映距离极值点的距离。
所以必须要考虑二阶微分。但计算二阶通常不容易，所以考虑怎么近似它。
我们发现，如果一个函数的一阶微分（的绝对值）普遍小于另一个函数的（普遍，少量点不能确定），则其二阶微分一般也更小。
所以大量点处的一阶微分，一般也能一定反应二阶微分的大小。所以用一阶微分来做分母。

- **Stochastic Gradient Descent（随机梯度下降）**
    常规梯度下降中使用的损失函数为 $L=\sum_{n}\large\left(\widehat y^n-(b+\sum w_ix_i^n)\right)$，每次考虑所有参数进行一次迭代：$\theta^{i+1}=\theta^{i}-\eta\nabla L(\theta^i)$。
    SGD 每次对单个样本进行一次迭代：顺序或随机选取一个样本$x^n$，令 $L^n=\large\left(\widehat y^n-(b+\sum w_ix_i^n)\right)$，然后 $\theta^{i+1}=\theta^{i}-\eta\nabla L^n(\theta^i)$。

因为只考虑一个样本，SGD 的单次迭代要快$n$倍，能更快收敛。
虽然单次迭代不一定向最优方向前进，但多次迭代后还是正确的，但收敛的路线会曲折。

SGD 在离最优点比较远时，收敛得比较快；而在接近最优点时，比较难收敛到最优点。

- **Mini-Batch Stochastic Gradient Descent（小批量随机梯度下降）**
    常用的方法。
    每次对少量个样本进行一次迭代。要更精确，同时也比较快。

- **Feature Scaling（特征缩放）**
    如果模型的特征$x_i$之间，值域相差很大，则调整参数$w_i$时，有些$w$的变化可能很小，但另一些$w$的变化可能非常大（与$x$的大小相匹配）。
    以两个特征为例，如果$x_1$很小，$x_2$很大，则$w_1$变化很大时，$w_2$可能变化依旧很小；$w_2$变化很小时，$w_1$可能变化非常大。导致损失函数关于$w_1,w_2$的图像可能像个很扁的椭圆。
    而如果$x_1,x_2$值域相近，$w_1,w_2$变化幅度相近，则图像会接近一个正圆。
    因为每次迭代是向切线走，但最优点一般在圆心，所以在椭圆上会需要更多次的迭代（切线偏离圆心），在正圆上只需要更少的迭代（切线指向圆心）。
    所以缩放至相近值域后，可能减少迭代次数，提高梯度下降的效率。

## 逻辑回归

### 线性回归

当问题总体上呈线性时，就可以使用线性回归，效果不好再试其它的。

线性回归模型为：$\widehat{y}=wx+b$，$x$是输入，$w$是权重，$b$是偏置。当$w,b$确定时，就得到了这个模型，对给定$x$可以预测/拟合结果。
$w,x,b$都可以是$d$维的（通常令$x_0=1$，那么$w_0$就是$b$），$d$是特征数量，$x$是该样本的每个特征值，$w$是该特征的重要性。

通过$m$个样本，计算预测结果与实际结果的误差的平方和（通常使用均方误差），可得到当前模型的误差/准确度。

线性回归模型是一个$d$维平面，将所有样本划分成两部分，并最小化预测结果与实际结果的误差和。





### 逻辑回归

逻辑回归是分类。用激活函数将连续函数的输出或模型$f(x)$（比如线性回归）转为离散（分界），预测每一类的概率。
逻辑回归模型：$g(f(x)) > 0.5$，其中$g$是激活函数。
使用 sigmoid 时，模型为：$\theta(f(x))=\frac{1}{1+exp(1-w^Tx)}$。

sigmoid 函数：$\theta(x)=\frac{1}{1+e^{-x}}$。值域为$(0,1)$，$\theta(0)=0.5$。

梯度更新：
对参数$w_j$，其一轮训练的梯度为：$\frac 1m\sum_{i=1}^m(\widehat y_i-y_i)x_{ij}$，其中$\widehat y_i$是对第$i$个样本的预测结果，$y_i$为实际结果，$x_{ij}$为第$i$个样本的第$j$个特征值。（使用交叉熵为损失函数）





> 1. 计算误差：将预测值与实际观测值进行比较，计算误差。对于二分类问题，可以使用交叉熵损失函数来度量误差。损失函数表示为：
>
>     $J(\theta) = -\frac{1}{m}\sum_{i=1}^{m}[y^{(i)}\log(h_\theta(x^{(i)})) + (1-y^{(i)})\log(1-h_\theta(x^{(i)}))]$
>
>     其中，$y^{(i)}$ 是第 i 个样本的实际观测值。
>
> 2. 计算梯度：对于每个参数 $\theta_j$，计算损失函数关于该参数的偏导数。梯度表示为一个与参数维度相同的向量，记为 $\nabla J(\theta)$。对于逻辑回归模型，梯度的计算可以使用以下公式：
>
>     $\frac{\partial J(\theta)}{\partial \theta_j} = \frac{1}{m}\sum_{i=1}^{m}(h_\theta(x^{(i)}) - y^{(i)})x_j^{(i)}$
>
>     这里，$x_j^{(i)}$ 表示第 i 个样本的第 j 个特征值。梯度表示了损失函数在参数空间中的变化方向和速率。
>
> 3. 更新规则：使用梯度下降算法来更新参数。一般的更新规则是将参数减去学习率乘以梯度。学习率控制了参数更新的步长。对于参数 $\theta_j$ 的更新规则可以表示为：
>
>     $\theta_j = \theta_j - \alpha \frac{\partial J(\theta)}{\partial \theta_j}$
>
>     其中，$\alpha$ 是学习率。

> 正则化系数：
> 通过增加 C*$\theta_j$/m 到梯度中，可以对参数更新进行正则化，控制参数的大小和模型的复杂度，从而提高模型的泛化能力和鲁棒性。
>
> 1. C = 0：这相当于没有应用正则化，模型将只考虑原始的损失函数，可能导致过拟合问题。在某些情况下，如果数据集很小或者特征已经经过了特征选择等处理，可以考虑不使用正则化。
> 2. 较小的 C 值：当数据集较大或特征较多时，可以尝试较小的正则化系数，例如 C = 0.01 或 C = 0.1。较小的正则化系数可以允许模型有更多的自由度，更多地拟合训练数据，但可能导致模型对噪声敏感。
> 3. C = 1：这是一个常见的选择，相当于对正则化和原始损失函数给予相等的权重。C = 1 可以被视为一种平衡的选择，可以在一定程度上控制过拟合问题。
> 4. 较大的 C 值：当数据集较小、特征较少或者存在较多的噪声时，可以尝试较大的正则化系数，例如 C = 10 或 C = 100。较大的正则化系数可以对参数施加更强的约束，减少模型对训练数据的过拟合。

## CNN


**全连接层** (fully connected layer, FC)

> 全连接层聚合之前卷积层或池化层提取的特征，便于进行分类。
> 全连接的拓扑结构可以让网络学习到输入和输出之间的复杂关系。通过全连接层，网络可以学习出更高层次的特征表示。
>
> 全连接层实际上是把 feature map 聚合成一个值（将分散的特征表示映射到样本标记空间？），减少特征所在位置对分类的影响：不管特征在哪里，只要这个特征值大，就会统计认为该特征存在
> 因为一定程度忽略了空间结构特性，所以全连接层不适用于在方位上找 pattern 的任务。

全连接层的每个神经元会执行一次$W\vec x+B$，$\vec x$是前一层是所有神经元，$W, B$是该神经元的学习参数，因此全连接层的参数数量为$(全连接层神经元数量+1)*前一层神经元数量$。

全连接层参数极多且难以训练，因此有以下方式减少或避免全连接层：

- 用 1*1 卷积层代替全连接层聚合学习出的特征。
- 用全局平均池化 (global average pooling, GAP) 代替全连接层聚合学习出的特征。
- 逐渐减小数据量。

### 激活函数

泰勒公式：许多函数可以用 N 次多项式近似。
通过不断叠加网络层，可以得到一个复杂的神经网络函数。

线性模型只能解决线性分类问题。因为线性函数具有可加性和齐次性，所以组合多个线性函数依然是线性模型。
激活函数可以将线性函数转为非线性函数，从而得到非线性模型。

激活函数的输出称为激活值/活性值 (activations)。

**非线性**

激活函数为网络引入非线性，从而可以让一个拥有单层足够多隐藏节点的浅层网络，理论上能拟合任意一个函数。

[以 Relu 为例](https://www.zhihu.com/question/1942676565583437862/answer/1944492319920349460)，虽然单个 Relu 神经元的输出只是带一个拐点的折线，但 n 个 Relu 带权加在一起就可以形成 n 个拐点的折线即分段线性函数，只要 n 足够大就可以逼近所有连续函数。

**softmax**

softmax 可以让大的更大、小的更小，拉开更可能与更不可能的概率差距。

当 x 的数值分布越不均匀时，由于$e^{\max(x_i)}$与`\sum_{i=1}^nx_i`会更接近，导致`softmax(\max(x_i))\to 1`，且`softmax(\min(x_i))\to 0`。





### MLP

多层感知机 (*MLP*, multilayer perceptron) 就是若干层全连接层的堆叠。
最后一层是输出层；除最后一层外的每层全连接层都使用非线性激活函数引入非线性，其中每一层称为隐藏层，其输出称为隐藏表示/隐藏层变量/隐藏变量 (hidden representations, hidden variable)，可表示为 $h^{(l)}=f(W^{(l)}h^{(l-1)}+b^{(l)})$（$f$ 是激活函数）。

- 一层 encoder layer 包括：multi-head self-attention，Add & Norm，FFN，Add & Norm。
- 一层 decoder layer 包括：multi-head mask-attention，Add & Norm，multi-head cross-attention，Add & Norm，FFN，Add & Norm。
- 模型包含多层 encoder/decoder layer。
- 对一个 query，只需要执行 encoder 一次获取 K, V，但要执行 decoder 若干次：对每次输出都产生一个新的 Q，与之前的 K, V 一起输入到 decoder 得到下一个输出，直到得到结束符。
- enc-dec 中 encoder 的输出是 K, V，是必须要存储的，所以它和 KVCache 无关。

**输入转为 embedding**

1. token embedding：输入首先通过 vocab_size 大小的字典将每个 token 转为 d 维向量（比如通过 `nn.Embedding`，实际就是 gather）。字典是 vocab_size * d 的可学习的 embedding 矩阵。
2. positional encoding：为序列添加位置信息：通过绝对位置编码公式或可学习的位置嵌入矩阵（字典）等方式，生成每个位置的位置信息 embedding。
3. 将上述两者相加，合并 token 和位置信息得到输入。

**Position Embedding**

可以发现 attention 只是算了单词之间的相关性，不会考虑任何位置信息，因此对一个输入 X，其结果会与打乱顺序后的 X 完全相同，即丢掉了序列顺序信息。
因此对输入进行 attention 计算前，需要先加上位置信息 position embedding。

把单词的词向量和位置向量进行叠加，这种方式就称作位置嵌入。



### encoder

Encoder block 结构为 Multi-Head Attention + Add & Norm + Feed Forward (FFN) + Add & Norm。
Encoder block 接收输入矩阵 $X_{n\times d}$，输出矩阵 $O_{n\times d}$。

Encoder 由多个 Encoder block 叠加组成，最终输出编码信息矩阵 C，它会和第一层之外的**每一层** Decoder 进行结合（是除第一层外每一层 decoder 的 K, V 输入）。

编码器负责理解和提取输入文本中的相关信息，并利用 self-attn 理解文本中的上下文关系。

**Attention**

Attention 机制来源于人类的视觉注意力机制：人类视觉通过快速扫描全局图像，获得需要重点关注的目标区域，也就是一般所说的注意力焦点，而后对这一区域投入更多注意力资源，以获取更多所需要关注目标的细节信息，而抑制其他无用信息。

简单来说，通过一个 Query（查询变量）从一堆 Key-Value 表示的信息（Key 是信息的地址/键，Value 是信息的内容）中找到对于 Query 而言重要的信息，以此获取更多对于 Query 而言更重要的细节信息，抑制其他无用信息。

> **QKV**
>
> 类似检索，将生成词向量视为用当前的 word 作为 query，把整个序列当做一个 key-value 数据库，然后去模糊查询。
> Q 会与 K 进行点积计算相关性（结果越大，表示 Query 和 Key 的相似度越高），然后用相关性作为系数乘 V 得到输出。
>
> 不同的 Q 会注重于不同的 K，得到更相关的 V。

Self-Attention 捕获了同一个句子中单词之间的一些句法特征（有一定距离的短语结构），且无论句子序列多长，都可以很好地提取句子特征，且既可以提取句法特征也可以提取语义特征。这解决了 RNN 序列长距离依赖的问题。
并且对于一个句子而言，每个单词的计算是可以并行的。

**Self-Attention**

设 token 数量为 N。

首先将输入 X（单词的表示向量 x 组成的矩阵或上一个 Encoder block 的输出）经过与三个权重矩阵的三次乘加 $X*Q_W+Q_B,\ X*K_W+K_B,\ X*V_W+V_B$ 得到 Q, K, V（三者的行数与 X 一致，即每行代表一个 token），然后计算：

$$Attention(Q,K,V)=\text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

$QK^T$ 的每个元素是 Q 的行向量与 K 的列向量的点积，大小为 N*N。该矩阵可以表示输入 token 之间的注意力强度。

> 生成 QKV 的线性变换层称为 Query-Key-Value Projection。
>
> 这个 attention 也称为 Scaled Dot-Product Attention（SDPA，缩放点积注意力），因为是对 Q, K 做点积， 再除以$\sqrt{d_k}$缩放，然后进行 softmax。
>
> softmax 的维度是 attn score 的最后一维，decode 中对 self 是已生成长度 (step)，对 cross 是 encoder_seq_len。
>
> $d$是 Q, K 的列数/向量维度，除以$d_k$的平方根是为了防止结果过大、分布过集中：假设 Q, K 的元素均值为 0、方差为 1，那么 $QK^T$ 的元素均值为 0、方差为 d。当 d 很大时，内积的方差也会很大，结果绝对值会在若干倍的$d$之间；由于 d 通常至少为 64，$e^{kd}$将非常大，$e^{-kd}$将非常小，会导致 softmax 几乎将全部的概率分布都分给了最大值对应的标签，使该维度的数量级较大，进而导致后续求梯度时 softmax 的影响会消失。除以 $\sqrt{d_k}$ 后内积的方差会变为 1，softmax 的分布也不会特别集中，降低结果的数量级。
>
> $QK^T$ 就是单词之间 (word2word) 的 attention map，即 attention 来源于 source 与 source 本身，所以叫 self-attention。

然后对矩阵的每一行进行 Softmax，得到每个单词与其它每个单词的 attention 比例。

然后与 V 相乘，得到结果 Z。单词 i 会用与每个单词 j 的 attention 比例 * $Z_j$、最后求和来计算 i 的最终结果。

**Multi-Head Attention**

多头注意力包含多个 Self-Attention 层：首先将输入 X 分别传给 h 个不同的 Self-Attention，计算得到 h 个输出矩阵Z1~Zh，然后将这些结果 concat 拼起来，再传入一个的 linear 线性变换层（乘 hidden*hidden 的矩阵然后 + bias）得到最终结果。
最后的 linear 将单纯拼接后的多个空间的输出融合，映射回原本的空间。

> concat + linear 称为 输出投影层 (out_proj)，将多头注意力输出的结果映射回模型的主维度。
>
> 每个 head 会均分 hidden size 的长度，最后拼接后再乘线性层恢复回去。假设 hidden size 是 768，12 个 head，则每个 head 的 hidden size 称为 d_h / dim_per_head / head size = 768/12=64，QKV 矩阵的大小是 768*64。
>
> 相比单个 attention，如果使用 multi-head attention，还需要在所有 attention 输出后接一个 concat + linear，即需要额外的参数。

多层是为了捕获单词之间多种维度上的相关系数 (attention score)。
多套参数相当于把原始信息放入了多个子空间中，使 attention 可以注意到不同子空间的信息，捕捉到更加丰富的特征信息。

**Add & Norm**

就是残差与归一化层：设输入为 X，则输出为 $\text{LayerNorm}\left(X+f(X)\right)$。
$f(X)$ 是前一层的输出，在 encoder 中为 MultiHeadAttention(X) 或 FeedForward(X)。

$$\text{LayerNorm(x)}=\frac{x-\mu}{\sqrt{\frac1d\sum_{i=1}^n(x_i-\mu)^2+\epsilon}}$$

可能包含可训练的权重和偏置参数：

$$\text{LayerNorm(x)}=\gamma\cdot\frac{x-\mu}{\sqrt{\sigma^2+\epsilon}}+\beta$$

> 常规的归一化同时用均值和方差进行中心化和缩放，而 RMSNorm (Root Mean Square Layer Normalization) 仅用均方根 (RMS) 缩放（不用方差），不中心化（无均值减法）：$\text{RMS}(x)=\sqrt{\frac1d\sum_{i=1}^nx_i^2},\ \text{RMSNorm}(x)=\frac{x}{\text{RMS}(x)+\epsilon}$。
>
> RMSNorm 无需计算均值，计算复杂度更低；保持均值非零而不强制对齐到零，缓解了梯度消失问题（LayerNorm 强制均值为0、方差为1），一般表现接近或更好。LLM 中常用 RMSNorm 进行正则化。

> 归一化将不同量纲、不同范围的数据转换到统一的标准下以便进行比较。
>
> 残差可以缓解梯度消失，保证数据特征分布的稳定，加速模型收敛。还可以让网络只关注当前差异的部分？

**Feed Forward**, **Feedforward Neural Network Layer**, **FFN**

Feed Forward 层 (FFN, FFNN, 前馈神经网络层) 通常由两层全连接层（线性层）和一个非线性激活函数组成 (Linear + Relu + Linear)：第一层的激活函数为 Relu/Gelu 等，第二层不使用激活函数：$FeedForward(X)=\max(0,\ XW_1+b_1)W_2+b_2$。
（就是一个两层的 MLP）

第一层全连接层会升维，把输入从当前维度 h 映射到通常为 4h 的中间层，第二层全连接层负责降维，把 4h 的中间层结果映射到初始维度 h。即 $W_1$ 维度是 $h*4h$，$W_2$ 维度是 $4h*h$。

> 升维的维度即 ffn hidden size 称为 intermediate size，不同模型可能不同。
>
> **为什么 FFN 要先升维再降维？**
>
> [解释有很多种](https://www.zhihu.com/question/665731716)，比如：
> 设 hidden size 是 M，$W_1,W_2$ 维度分别为 $M*N,\ N*M$。
> 一个 token 会存在于 M 维空间中，$W_1$ 会将 token 投影成 N 个激活值，经过 relu 后假设有 k 个激活值非 0，则乘 $W_2$ 后的 token 存在于最多 $k\le N$ 维空间中（乘 $W_2$ 是对 $W_2$ 的每列的固定 k 个数线性组合），如果 $N$ 太小即 $k$ 也很小，relu 就很可能导致重建后的 token 落入 M 维空间的一个低维子空间中，降低 token 的表达能力。
> $k<M$ 的概率即为 token 降秩的概率，应尽可能低。假设输出均匀即每个位置有 1/2 概率为 0，则 N = 4M 是比较小但效果够好的取值。
>
> 没太懂，会使矩阵降秩是指什么？

**Linear**

线性变换层是一个简单的全连接神经网络 $F=WX+B$，它将解码器产生的向量 A 投影到一个更高维度的向量 B 上。
如果模型的词汇表是 N 个词，那么向量 B 就有 N 维，每个维度对应一个唯一的词的得分。
然后 B 经过 softmax 得到每个词的概率后即可得到最终要输出的词。

encode 中没有 linear，decoder 中有。

**lm_head**

decoder softmax 前的 linear 将隐藏向量 (维度为 hidden size) 转换为词汇表大小 (如 50,000+) 的 logits，这一层称为 lm head (language modeling head)，通常是 `nn.Linear(hidden_size, vocab_size)`。



### decoder

> https://www.zhihu.com/question/588325646

Decoder block 结构为 Masked Multi-Head Attention + Add & Norm + [Multi-Head cross Attention + Add & Norm] + Feed Forward + Add & Norm，最后为 linear + softmax。

Decoder 由多个 Decoder block 叠加组成。

- 与 Encoder 相比，Decoder block 使用 Masked Multi-Head Self-Attention，其 K, V 也使用输入进行计算，只是有一个 mask 矩阵。
- 第二个及后续的 Multi-Head Attention 的 K, V 使用 Encoder 的编码信息矩阵 C 进行计算，而 Q 仍使用上一个 decoder block 的输出计算，这种 Q 与 KV 来源不一致的称为 *cross attention*。
  只有 encoder-decoder 有该层和对应的 Add & Norm，decoder-only 没有（即 decoder-only 的 decoder block 会少一个 attention 和归一化）。
- cross-attention 中输入 token 之间不会进行 attention（没有 self）；输入 token (Q) 称为 tgt/target，encoder 的输出 (KV) 称为 src。
- enc-dec 中第一个 mask attention 用于处理已生成部分的信息，后续 cross attention 用于关注 encoder 的输入信息。
- Decoder block 最后有一个 Softmax 来预测下一个单词。

**mask**

mask 矩阵就是 N\*N 的 01 矩阵，会在 Softmax 前与输入进行相乘，使第 i 个 token 在 i + 1 ~ n 位置的 attention 结果变为 0，然后与 V 相乘时就可以忽略后面单词的信息，防止后续输入干预当前输出的结果。

$$MaskAttention(Q,K,V)=\text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}+M\right)V$$

> 只与它前面的 token 交互来计算的 attention 称为 *masked attention*/*causal attention*。
>
> 为什么做 mask 还有其它的解释：让训练阶段和测试阶段行为一致，不会出现间隙，避免过拟合：
>
> - 训练时输入 decoder 的是正确输出的 embedding，包含所有词，如果不 mask 后续的词就会对前面的预测产生影响。
> - ~~测试（推理？）时没有预设的输入，只能随机初始化一个输入 embedding，因此不管是否 mask，在第 i 次输入时 i 后面的词都不会对 i 产生影响，因为 i 后面的词还没有输出出来。~~ 推理时，如果没有 KV cache 应该也要 mask 避免 token 1 去算预测出来的 token2 的 attention，来让结果和之前一致；有了 KV cache 每次只算要预测的 token i 这一行，i 后面的都还没有，所以是否 mask 也无所谓了。

**mask 实现**

- 如果没有 kv cache：第一轮 decode 需要 mask；后续轮 decode 与第一轮无区别，只是多了一行，所以仍需要 mask。
- 如果有 kv cache：第一轮 decode 需要 mask；后续轮 decode 只需要算新的一行，不需要算历史输入，所以不需要 mask（新的 token 可以关注到过去所有的不会 mask；历史 token 的计算在 kv cache 中，已经是 mask 算了的）。
  如果第一轮 decode 的输入长度是 1，那同后续轮一样也不需要 mask。

cross-attention 没有 casual mask，但可能有 encoder_attention_mask 用来避免关注 encoder 输入中的 padding token。

**decoder prompt**

即使是 enc-dec，也可以在 decode 前设置输入 decoder prompt，相当于强制它作为 decoder 最开始的输出，从而引导生成格式、控制生成任务。
比如：`decoder_input = "<s>The summary is:"`可以使模型进行摘要，`<s>`是初始生成标记 (Start Token，表示生成的开始)，后面是条件性前缀。





### KV Cache

decoder 是自回归模型，每轮输出一个 token，然后将该 token 作为下一轮的输入继续推理。

在第 i 轮推理中，由于 mask 的存在，计算输出 $t_i$ 只需要将 $Q_i$ 与 $K_1,K_2,...,K_i$ 做内积、再与 $V_1,V_2,...,V_i$ 相乘就得到该 attention 新的输出，其中 $K_i=t_{i-1}W_K$（$V$ 同理）会在后续每轮计算都用到，因此可以缓存 $K_{1\sim i},V_{1\sim i}$（第 i 轮计算时追加一个 $K_i,V_i$ 到 cache）。

缓存后每次计算 K, V 时只需计算新的一列，而非一整个矩阵。

> encoder-decoder 应该只需要缓存第一个 decoder 的 $K_{1\sim i},V_{1\sim i}$ 和 encoder 的输出？因为后续 decoder 的 KV 来自 encoder。
> decoder-only 应该是需要缓存所有 decoder 的 KV？因为第一个和后续 decoder 是一样的，KV 来自最新的输入。





### 其它

**prefill**, **decode**

LLM 推理可以分为 prefill (预填充) 和 decode (解码) 两阶段：

- prefill：对用户输入中的所有 token 计算 QKV (encoder)，然后输入一次 decoder 得到第一个输出 token。
  具体包括：将输入转为 token、前向传播计算、缓存最终的 KV 结果、计算第一个 token。
  - 每个 token 的 QKV 及 attention 计算都是可以并行的。
  - 由于有 attention，所以复杂度为 O(n^2)（n 是输入序列长度）？
  - 因为包括完整的矩乘，所以是计算密集的。
- decode：多次执行，逐 token 地生成输出序列。
  - 计算是串行的。
  - 与 prefill 有所不同：随着输出序列长度增加，attention 计算复杂度将大幅增长（平方级），对长序列的推理直接计算代价很大。但可以复用已生成的 KV 结果，不需重新计算，每次只需要计算 Q 的最后一行 qKV 而非 QKV 的 attention，这样复杂度降低了一个量级。
  - 优化后是向量乘，所以是访存密集的。

**不同架构**

- **encoder-only**：注重理解输入的内容、语义信息提取，擅长对文本内容进行分析、分类。
- **decoder-only**：适合生成类。同样擅长对文本内容进行分析、分类（因为与 encoder 相比只是多了 mask），但会确保位置 i 的输出只依赖于位置 i 之前的已知输出。
- **encoder-decoder**：输出强烈依赖输入、输入和输出序列之间存在复杂映射关系的，比如翻译和文本总结。

> enc-dec 会先将输入放入 encoder 生成一组中间结果，在 decoder 中与其计算 cross-attention。
> decoder-only 没有 encoder，会将输入直接放入 decoder，只会计算 mask attention。
>
> enc-dec 在每层 decoder 都能很好的利用输入的信息，所以很适合翻译；dec-only 适合续写，但翻译效果也可以所以现在很少用 enc-dec。
>

- enc-dec 难以扩展、训练效率低，由于 scaling law，即使 enc-dec 效果略好，用它更高的训练成本去训练 decoder-only（扩大其模型或训练更多样本）效果反而可能更好。

**teacher forcing**

Teacher Forcing 是一种用于序列生成任务的训练技巧，与推理时的 Autoregressive 模式相对：

- Autoregressive 模式下，t 时刻 decoder 的输入是模型自己 t-1 时刻的输出。
- Teacher-Forcing 模式下，t 时刻 decoder 的输入是 Ground-truth 语句中 t-1 位置的实际 token；模型 t-1 时刻的输出只用来计算误差、更新模型，不作为后续输入。

在序列生成中，推理时需要不断将模型输出重新输入给模型，如果这样训练，在模型训好前可能会导致误差越来越大；使用正确结果喂给模型进行训练，能矫正模型的预测，加快收敛速度，且是保证 Transformer 模型能够在训练过程中完全并行计算所有 token 的关键技术？但也存在很多缺点。

**exposure bias**

Exposure Bias 是 Teacher Forcing 训练会产生的问题：由于训练和预测时 decode 行为不一致， 预测单词 (predict words) 在训练和预测时是从不同的分布中推断出来的。这种不一致导致的训练模型和预测模型的 gap，就叫 Exposure Bias。

forcing 强制纠正每次错误的输出，可能会导致模型过分依赖邻近位置的信息，而忽略更远地方的信息。

**shared embedding**

encoder-decoder 结构中 encoder 和 decoder 的 embedding 参数可以共享（使用同一套参数），包括：Token Embeddings（将输入词映射为向量）和 Positional Embeddings（为序列中的位置信息编码）。
共享参数可以在词表较大时减少模型参数量，避免编码器和解码器表示空间的不一致。

> 似乎只是 token emb 共享，position emb 不共享？

LM Head 的权重矩阵也可与输入端的 Token Embedding 矩阵共享参数，后者是将词映射到向量空间，而 LM Head 将向量空间映射回词的概率分布，两者具有对称性。计算时 $W_{head}=W_{emb}^T$。
同样可以减少模型参数量，使输入输出共享同一语义空间。
如果输入和输出的词汇分布差异大（如多语言任务），或下游任务中需要调整输出分布（如添加新词），则可能不适合共享。如果是图像描述生成（输入是图像特征，输出是文本），输入无嵌入层，也无法共享。

**beam search**

> https://zhuanlan.zhihu.com/p/82829880
> https://zhuanlan.zhihu.com/p/28048246

用于序列生成。
比如 transformer 每次计算完后，可以选择概率最高的 token 作为下一个 token 作为输入，也可以选 k 个。

比较简单的方法是贪心：每次生成下个 token 时，选择当前概率最大的 token 作为输出。
贪心只按局部最优做出选择，往往得不到全局最优解。

beam search 就是扩大搜索空间的贪心，是贪心和穷举的折中：bs 有一个超参数 beam size (束宽)，设为 k。
第一次生成 token 时，选择当前概率最大的 k 个 token 保留；后续输出时，用这 k 个 token 构造出 k 个输入，然后从这些输入的所有可能输出中再次选择概率最大的 k 个保留（每个输入保留一个，还是没有该限制？）。最终也会得到 k 个输出。

**Bert**

> BERT 的意义在于：从大量无标记数据集中训练得到的深度模型，可以显著提高各项自然语言处理任务的准确率。
>
> GPT：GPT 使用 Transformer Decoder 作为特征提取器、具有良好的文本生成能力，然而当前词的语义只能由其前序词决定，并且在语义理解上不足（单向编码）。
> BERT：使用了 Transformer Encoder 作为特征提取器，并使用了与其配套的掩码训练方法。虽然使用双向编码让 BERT 不再具有文本生成能力，但是 BERT 的语义信息提取能力更强。
>
> 单向编码只会考虑上文来预测下一个词，双向编码会同时考虑上下文的信息。

Bert 模型会在句子的最开始加一个 [CLS] token 作为输入。在多层 encoder 中，因为每一层的自注意力机制都会处理、考虑整个输入序列的所有 token，所以 [CLS] 的输出表示会逐层累积整个句子的上下文信息，其最终输出表示可以包含整个句子的全局信息。通过在 [CLS] 标记的输出表示上添加一个全连接层，可以直接进行分类任务。
（对于句子情感分类，只需要拿特征当中的第0个词 [CLS] 的输出来进行分类）

在 mask 训练中，预测 masked token 是根据对应位置的结果决定的，而不是 CLS。
为每个 token 进行分类时，也是使用对应位置的输出而非 CLS。

> Bert 的 FFN 使用 Gelu 作为激活函数。



**Bert 无监督训练**

BERT 的训练分为两阶段：

1. 使用易获取的大规模无标签余料，训练基础语言模型。
2. 根据指定任务的少量带标签训练数据进行微调训练。

同时训练 MLM (Masked Language Model) 和 NSP (Next Sentence Prediction) 两个任务，来加强模型对语义的理解能力。

**预训练**

CNN 浅层提取的特征对于许多任务都是通用的。

假设有两个尽量相似的任务 A 和 B（如都为图像处理任务），任务 A 是目标任务，任务 B 是可以提前训练好的任务。
预训练就是先训练任务 B 得到一个 CNN 模型 B，由于 CNN 浅层特征的通用性，可以做两种处理：

1. fine-tuning（微调）：浅层参数使用模型 B 的参数，高层参数随机初始化，利用任务 A 训练模型 B，浅层参数也随着变化。
2. frozen（冻结）：浅层参数使用模型 B 的参数，高层参数随机初始化，利用任务 A 训练模型 B，但浅层参数一直不变。

预训练能加速相似任务训练的收敛速度，且可以缓解模型数据量小、难以训练的问题。

**LLM 性能指标**

- **TTFT**: Time To First Token, 首 token 延迟，衡量响应速度。
- **TPOT**: Time Per Output Token，每个输出 token 耗时，衡量生成吞吐效率。
- End-to-End (E2E) Latency: 请求从提交到收到最后一个 token 的耗时。
- TPS: Total Token Throughput，包括 prompt 与所有输出 token 的平均耗时。

> TBT (Time Between Tokens) 应该和 TPOT 一样。

[trtllm 中](https://nvidia.github.io/TensorRT-LLM/deployment-guide/deployment-guide-for-deepseek-r1-on-trtllm.html#key-metrics)有 TPOT 和 **ITL** (Inter-Token Latency)，两者都是输出 token 耗时，但统计方式和侧重不同：

- TPOT = n 个请求的 TPOT / n（每个请求的 TPOT 的平均值）。
  - 每个请求的权重相等，无论一个请求生成了 10 个还是 1k 个 token，它影响结果的程度与其它请求相同。
  - 反映用户感到的平均生成速度（用户体验）。
- ITL = n 个请求的输出 token 总耗时 / 总 token 数（总耗时 / token 数）。
  - 每个 token 的权重相等，但请求间不同：如果一个请求生成了 1k 个 token，则其权重是 10 个 token 的请求的 100 倍。
  - 更能反映服务器处理单个 Token 的平均耗时（整体吞吐效率）。
  - 如果系统在处理长序列时效率更高，则其 ITL 更低，这更能体现出优化的效果、系统的优势。
