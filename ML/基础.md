### 梯度下降

基本思想为：为了尽快找到函数的极值，选择往函数变化最快的方向前进；如果要找误差函数的极小值，那么应该朝函数的负梯度方向搜索，即往负梯度方向更新参数。
函数变化最快的方向就是函数的梯度/导数。

原理：泰勒公式：如果函数$f(x)$在含$x_0$的某个开区间$(a,b)$内有 n+1 阶导数，则对$\forall x\in(a,b)$，$f(x)=f(x_0)+\sum_{i=1}^n\frac{f^{(i)}(x_0)}{i!}(x-x_0)^i+R_n(x)$，其中余项$R_n(x)$是误差。

梯度下降实际上是对误差函数$f$的泰勒级数进行一阶展开，得到近似函数：$f(x)=f(x_0)+f'(x_0)(x-x_0)$，其中目前的模型为$x_0$。
然后我们希望找到新的模型参数$x$使得$f(x)$尽量小。
设新的参数$x=x_0+\lambda \vec{d}$，所以$f(x)=f(x_0)+\lambda \vec d\cdot f'(x_0)$，通过[这里的推导](https://zhuanlan.zhihu.com/p/152566066)可以得出，当$x=x_0-\lambda f'(x_0)$时，新模型的（拟合后的）误差最小。其中$f'(x_0)$就是梯度，$\lambda$是未知量，用学习率超参数表示。

Gradient Descent：$\theta^{i+1}=\theta^{i}-\eta\nabla L(\theta^i)$
$\eta$为步长或学习率，当步长小时，收敛慢；步长大时，容易越过极小值点，导致错误或收敛更慢。
因此通常随着迭代次数增加，减小步长。

Vanilla Gradient Descent：$$\theta^{i+1}=\theta^{i}-\frac{\eta}{\sqrt{i+1}}\nabla L(\theta^i)$$
随迭代次数的增加，减少学习率。

梯度下降有三种形式：

- **Batch Gradient Descend（批量梯度下降）**

每次训练，使用所有样本计算误差，然后更新参数。
可以直接收敛到最优解。
但每次迭代训练很慢；空间开销极大；只向误差函数梯度方向前进，不具备跳出局部最优解的能力。

- **Stochastic Gradient Descent（随机梯度下降）**

每次训练，使用单个样本计算误差，然后更新参数。
每次迭代更快；增加了随机性（仅通过单个样本计算误差，可能不准确，与最终的模型不符），有一定跳出局部最优解的能力。

收敛更慢：虽然单次迭代不一定向最优方向前进，但整体上（多次迭代后）还是正确的，但收敛的路线会曲折（噪声多）。
SGD 在离最优点比较远时，收敛得比较快；而在接近最优点时，比较难收敛到最优点。

- **Mini-Batch Stochastic Gradient Descent（小批量随机梯度下降）**

常用的方法。
每次训练，使用部分样本计算误差。
比较精确，同时也比较快；多样本的计算能够并行，利用硬件。

三种形式就是不同 batch size 的梯度下降，需要根据实际情况调整该值。

> 使用所有数据完成一次训练，称为一个 epoch。
> 因此 batch size 越小，进行一个 epoch 所需的迭代次数就越多，所花的时间就越多（需要更多次的梯度下降）。
>
> 增大 batch size 能够充分利用硬件并行计算的能力；进行一个 epoch 所需的时间少。
> 但当 batch size 过大时，内存不能容纳所有样本（或者 GPU 显存？），就无法进行高效计算（需要从硬盘或网络等其它介质读取）；每个 epoch 中的迭代次数少，因此想达到相同的精度，需要的 epoch 就更多。
>
> 因此，batch size 越大，进行一个 epoch 所需的时间越少，但需要的 epoch 越多。
> 所以 batch size 不能过大也不能过小。降低和提高 batch size 都有可能降低训练效果。
>
> 这与==数据的多样性==有关。
> 一轮迭代中，使用的数据的多样性越好、差异性越明显，效果越好。
> 如果使用的数据都差不多，那么用1个样本和用1000个样本计算出来的梯度是差不多的（梯度会除以样本数量），对模型的贡献差不多，后者多处理的样本就白算了，并不会提高训练效果。
> 因此如果有足够大的样本（容易取到多样的样本），或样本的质量好、差异性好，那么可以使用更大的 batch size；否则 batch size 就不需要多大。
> 简单来说，如果数据有 n 个类，那么 batch size 差不多取 10n，再大就很难了。
> （此外还取决于优化算法）

**P5 Where does the error come from?**

- bias：$f^*$的平均值距离实际$\widehat{f}$的偏差。
- variance：各预测函数 $f^*$ 距离它们平均值的方差。
- 越简单的 model 受样本数据的影响越小，得到的预测函数$f^*$的方差(variance)越小，但其平均值$\overline{f}$距离正确预测的偏差(bias)越大。
- 如果误差主要来自偏差，则 model 过简单(underfitting)；如果主要来自方差，则 model 过复杂(overfitting)。
- 如果模型与训练数据误差大，则偏差太大(underfitting)；如果模型与训练数据误差小，但与测试数据误差大，则方差太大(overfitting)。
- 偏差太大：增加特征；设计更复杂的模型。
- 方差太大：增加训练数据量；正则化(regularization)。
- N-fold Cross Validation

**P6 Gradient Descent**

- 梯度下降：$\theta^{i+1}=\theta^{i}-\eta\nabla L(\theta^i)$

- **Vanilla Gradient Descent**

随迭代次数的增加，减少 learning rate。
$$\theta^{i+1}=\theta^{i}-\frac{\eta}{\sqrt{i+1}}\nabla L(\theta^i)$$

- **Adagard Gradient Descent**

对每个参数$w$使用不同的 learning rate，且根据历史微分结果调整。
令学习率为 $\eta^i=\frac{\eta}{\sqrt{i+1}}$，对参数$w$的偏微分为 $g^i=\frac{\partial C(\theta^i)}{\partial w}$，过去所有偏微分的均方根(root mean square)为 $\sigma^i={\sqrt{\frac{1}{i+1}\sum_{i=0}^i(g^i)^2}}$。
$$w^{i+1}=w^i-\frac{\eta^i}{\sigma^i}g^i =w^i-\frac{\eta}{\sqrt{\sum_{i=0}^t(g^i)^2}}g^i$$

但这似乎与直觉不符：偏微分$g$越大，似乎代表离极值点越远，所以系数应该更大；但除数以$g$为分母，$g$越大系数会越小。
事实上，上述直觉仅在函数有一个参数时成立。当函数有多个参数时，$\frac{一阶微分}{二阶微分}$的大小才能一定反映距离极值点的距离。
所以必须要考虑二阶微分。但计算二阶通常不容易，所以考虑怎么近似它。
我们发现，如果一个函数的一阶微分（的绝对值）普遍小于另一个函数的（普遍，少量点不能确定），则其二阶微分一般也更小。
所以大量点处的一阶微分，一般也能一定反应二阶微分的大小。所以用一阶微分来做分母。

- **Stochastic Gradient Descent（随机梯度下降）**
    常规梯度下降中使用的损失函数为 $L=\sum_{n}\large\left(\widehat y^n-(b+\sum w_ix_i^n)\right)$，每次考虑所有参数进行一次迭代：$\theta^{i+1}=\theta^{i}-\eta\nabla L(\theta^i)$。
    SGD 每次对单个样本进行一次迭代：顺序或随机选取一个样本$x^n$，令 $L^n=\large\left(\widehat y^n-(b+\sum w_ix_i^n)\right)$，然后 $\theta^{i+1}=\theta^{i}-\eta\nabla L^n(\theta^i)$。

因为只考虑一个样本，SGD 的单次迭代要快$n$倍，能更快收敛。
虽然单次迭代不一定向最优方向前进，但多次迭代后还是正确的，但收敛的路线会曲折。

SGD 在离最优点比较远时，收敛得比较快；而在接近最优点时，比较难收敛到最优点。

- **Mini-Batch Stochastic Gradient Descent（小批量随机梯度下降）**
    常用的方法。
    每次对少量个样本进行一次迭代。要更精确，同时也比较快。

- **Feature Scaling（特征缩放）**
    如果模型的特征$x_i$之间，值域相差很大，则调整参数$w_i$时，有些$w$的变化可能很小，但另一些$w$的变化可能非常大（与$x$的大小相匹配）。
    以两个特征为例，如果$x_1$很小，$x_2$很大，则$w_1$变化很大时，$w_2$可能变化依旧很小；$w_2$变化很小时，$w_1$可能变化非常大。导致损失函数关于$w_1,w_2$的图像可能像个很扁的椭圆。
    而如果$x_1,x_2$值域相近，$w_1,w_2$变化幅度相近，则图像会接近一个正圆。
    因为每次迭代是向切线走，但最优点一般在圆心，所以在椭圆上会需要更多次的迭代（切线偏离圆心），在正圆上只需要更少的迭代（切线指向圆心）。
    所以缩放至相近值域后，可能减少迭代次数，提高梯度下降的效率。

## 逻辑回归

### 线性回归

当问题总体上呈线性时，就可以使用线性回归，效果不好再试其它的。

线性回归模型为：$\widehat{y}=wx+b$，$x$是输入，$w$是权重，$b$是偏置。当$w,b$确定时，就得到了这个模型，对给定$x$可以预测/拟合结果。
$w,x,b$都可以是$d$维的（通常令$x_0=1$，那么$w_0$就是$b$），$d$是特征数量，$x$是该样本的每个特征值，$w$是该特征的重要性。

通过$m$个样本，计算预测结果与实际结果的误差的平方和（通常使用均方误差），可得到当前模型的误差/准确度。

线性回归模型是一个$d$维平面，将所有样本划分成两部分，并最小化预测结果与实际结果的误差和。





### 逻辑回归

逻辑回归是分类。用激活函数将连续函数的输出或模型$f(x)$（比如线性回归）转为离散（分界），预测每一类的概率。
逻辑回归模型：$g(f(x)) > 0.5$，其中$g$是激活函数。
使用 sigmoid 时，模型为：$\theta(f(x))=\frac{1}{1+exp(1-w^Tx)}$。

sigmoid 函数：$\theta(x)=\frac{1}{1+e^{-x}}$。值域为$(0,1)$，$\theta(0)=0.5$。

梯度更新：
对参数$w_j$，其一轮训练的梯度为：$\frac 1m\sum_{i=1}^m(\widehat y_i-y_i)x_{ij}$，其中$\widehat y_i$是对第$i$个样本的预测结果，$y_i$为实际结果，$x_{ij}$为第$i$个样本的第$j$个特征值。（使用交叉熵为损失函数）





> 1. 计算误差：将预测值与实际观测值进行比较，计算误差。对于二分类问题，可以使用交叉熵损失函数来度量误差。损失函数表示为：
>
>     $J(\theta) = -\frac{1}{m}\sum_{i=1}^{m}[y^{(i)}\log(h_\theta(x^{(i)})) + (1-y^{(i)})\log(1-h_\theta(x^{(i)}))]$
>
>     其中，$y^{(i)}$ 是第 i 个样本的实际观测值。
>
> 2. 计算梯度：对于每个参数 $\theta_j$，计算损失函数关于该参数的偏导数。梯度表示为一个与参数维度相同的向量，记为 $\nabla J(\theta)$。对于逻辑回归模型，梯度的计算可以使用以下公式：
>
>     $\frac{\partial J(\theta)}{\partial \theta_j} = \frac{1}{m}\sum_{i=1}^{m}(h_\theta(x^{(i)}) - y^{(i)})x_j^{(i)}$
>
>     这里，$x_j^{(i)}$ 表示第 i 个样本的第 j 个特征值。梯度表示了损失函数在参数空间中的变化方向和速率。
>
> 3. 更新规则：使用梯度下降算法来更新参数。一般的更新规则是将参数减去学习率乘以梯度。学习率控制了参数更新的步长。对于参数 $\theta_j$ 的更新规则可以表示为：
>
>     $\theta_j = \theta_j - \alpha \frac{\partial J(\theta)}{\partial \theta_j}$
>
>     其中，$\alpha$ 是学习率。

> 正则化系数：
> 通过增加 C*$\theta_j$/m 到梯度中，可以对参数更新进行正则化，控制参数的大小和模型的复杂度，从而提高模型的泛化能力和鲁棒性。
>
> 1. C = 0：这相当于没有应用正则化，模型将只考虑原始的损失函数，可能导致过拟合问题。在某些情况下，如果数据集很小或者特征已经经过了特征选择等处理，可以考虑不使用正则化。
> 2. 较小的 C 值：当数据集较大或特征较多时，可以尝试较小的正则化系数，例如 C = 0.01 或 C = 0.1。较小的正则化系数可以允许模型有更多的自由度，更多地拟合训练数据，但可能导致模型对噪声敏感。
> 3. C = 1：这是一个常见的选择，相当于对正则化和原始损失函数给予相等的权重。C = 1 可以被视为一种平衡的选择，可以在一定程度上控制过拟合问题。
> 4. 较大的 C 值：当数据集较小、特征较少或者存在较多的噪声时，可以尝试较大的正则化系数，例如 C = 10 或 C = 100。较大的正则化系数可以对参数施加更强的约束，减少模型对训练数据的过拟合。

## CNN



**激活函数**

泰勒公式：许多函数可以用 N 次多项式近似。
通过不断叠加网络层，可以得到一个复杂的神经网络函数。

线性模型只能解决线性分类问题。因为线性函数具有可加性和齐次性，所以组合多个线性函数依然是线性模型。
激活函数可以将线性函数转为非线性函数，从而得到非线性模型。



**全连接层** (fully connected layer, FC)

> 全连接层聚合之前卷积层或池化层提取的特征，便于进行分类。
> 全连接的拓扑结构可以让网络学习到输入和输出之间的复杂关系。通过全连接层，网络可以学习出更高层次的特征表示。
>
> 全连接层实际上是把 feature map 聚合成一个值（将分散的特征表示映射到样本标记空间？），减少特征所在位置对分类的影响：不管特征在哪里，只要这个特征值大，就会统计认为该特征存在
> 因为一定程度忽略了空间结构特性，所以全连接层不适用于在方位上找 pattern 的任务。

全连接层的每个神经元会执行一次$W\vec x+B$，$\vec x$是前一层是所有神经元，$W, B$是该神经元的学习参数，因此全连接层的参数数量为$(全连接层神经元数量+1)*前一层神经元数量$。

全连接层参数极多且难以训练，因此有以下方式减少或避免全连接层：

- 用 1*1 卷积层代替全连接层聚合学习出的特征。
- 用全局平均池化 (global average pooling, GAP) 代替全连接层聚合学习出的特征。
- 逐渐减小数据量。
