## 概念



**topk** **topp** 采样

Top-k 采样 (Top-k Sampling) 只保留概率最高的 k 个词（将其他所有词的概率设为0），然后在这 k 个词上重新归一化概率，按概率加权采样。

- 避免从大量低概率、不相关的词中采样，从而提高生成质量。
- k 是固定值，可能不适合所有场景或输入。

Top-p 采样 (Top-p / Nucleus Sampling) 动态地选择最小的词集合（即概率最高的部分词），使它们的累积概率超过阈值 p（例如 p=0.9）。然后在这个集合上重新归一化并按概率采样。

- 比 Top-k 更灵活。它能根据模型对当前上下文的“自信程度”动态调整候选集的大小。
- 在模型很确定时（少数几个词概率很高），候选集会很小；在模型不确定时（很多词概率都差不多），候选集会变大，保留更多可能性，从而增加多样性。

## 结构



### 绝对位置编码

**绝对位置编码** 为序列中的每个位置生成一个唯一的、固定维度的向量（位置编码），然后将这个向量直接加到 token 的 embedding 上。

位置编码方式：

- 使用正弦和余弦函数来生成不同频率的编码，即正弦位置编码 (sinusoidal positional embeddings)，以捕捉绝对位置和相对位置信息。
  
- 使用可学习 emb (learned embeddings) 将位置索引 (position_id) 转为相应的 emb。

问题：

- 由于编码是为特定长度范围设计的，当模型遇到比训练时更长的序列时，性能会明显下降，因为位置编码在训练范围外没有意义。或输入长度必须在指定长度内。
- 缺乏显式的相对位置建模：虽然正弦编码理论上能捕捉相对位置，但模型需要自己从加法操作中学习这种关系。



### RoPE, Rotary Embedding

**旋转位置编码** (**RoPE**, **rotary position embedding**) 利用向量的旋转变换来注入位置信息。

常规的编码是将 pos + token 的 emb 转为 Q, K，再计算 attention。
RoPE 不直接修改 token emb，而是在计算 attention score 前，对 Q 和 K 进行旋转变换。即不修改输入，而是直接在 attention 中加入了位置信息。
它将 Q 和 K 向量分组为二维子空间，然后根据其位置索引，应用旋转矩阵。这个旋转矩阵由正弦和余弦函数构成。

实现：

- 将 QK 的 d 维特征空间（d 维向量）两两分组（0 与 1，2 与 3...），得到 d/2 组二维向量。
- 对每个二维向量 i (x1, x2) 进行2D旋转，旋转角度与 token 在序列中的位置即组号 i 相关，为 $\theta_i=10000^{-2i/d}$，使得注意力点积结果变为相似度计算中 token 相对距离的函数？
- 二维向量逆时针旋转只需要左乘一个旋转矩阵。对所有 d/2 个二维向量旋转，可以乘一个大的稀疏旋转矩阵（效率低），也可分别相乘再拼接，见：https://zhuanlan.zhihu.com/p/718700160、https://zhuanlan.zhihu.com/p/642884818

原理与意义：

- 相似的向量在不同的子空间中也应该保持相似性，所以相对应的两个向量在子空间中被旋转某个角度后，可以再比较相似性。这样增强了注意力机制的鲁棒性。
- 融合了 token 的相对位置信息：
  - 设两个子向量 q, k 被旋转 $\theta_q, \theta_k$ 度得到 q', k'，它们点积的结果会与两个向量的夹角有关，这个夹角就取决于这两组 token 在序列中的相对位置。
    即这个编码函数是 q, k 和它们之间相对距离的函数。
  - 随着相对距离增加，q', k' 的角度会变大，它们的点积结果即相似度会变小。这实现了长距离衰减 (Long-term decay)，即两个相距较远的 token 的 attention score 会逐渐减小。

经过RoPE变换后，q和k的点积（即注意力分数）可以分解为一个仅依赖于相对位置 |m-n| 的函数。这意味着，无论两个词元在序列中的绝对位置是多少，只要它们之间的距离相同，它们的注意力交互模式就是一致的。

## Qwen3

> https://zhuanlan.zhihu.com/p/1970950251234963511



- 与 Qwen2 的不同：Qwen3 移除了 QKV-bias（q/k/v linear 后不加 bias），并在注意力机制中引入了QK-Norm（在 q/k-linear 后加入了 RMSNorm）。
- MLP 部分与 qwen2 一样，使用 *SwiGLU*。



**SwiGLU**

不是对整个输入向量应用同一个激活函数，而是让一部分输入作为门控，控制另一部分输入的“通过程度”。

`SwiGLU(x) = (xW + b) * Swish(xV + c) * W_down`

1. 输入 x 通过两个不同的权重矩阵 W 和 V（以及对应的偏置 b 和 c）进行线性变换，得到两个同样维度的向量：`a = xW + b`、`g = xV + c`。
   - W, V 的 shape 为 (hidden_size, intermediate_size)，intermediate size 可为固定值，对 qwen 来说是 3 * hidden_size（比传统 FFN 4h 更小）。
   - 分别称为 up-linear 和 gate-linear。
2. 向量 g 通过 Swish 激活函数。
   - Swish(x) = x * σ(βx)，其中 σ 是 Sigmoid 函数。在 SwiGLU 中，通常 β = 1，因此 Swish(x) = x * σ(x)。
3. 将未激活的向量 a 与作为"门"的 Swish(g) 进行逐元素相乘：`act = a * Swish(g)`。
4. 将高维输出投影回原大小：`output = act * W_down`。
   - 称为 down-linear。

但是比传统 FFN 参数量更高（多一个门控矩阵）、计算量更高（多一个矩乘和逐元素乘（这个可以忽略））。

意义：

- 更强的表达能力：门控机制允许模型学习更复杂的特征交互。门 Swish(g) 可以动态地、有选择性地放大或抑制输入中不同维度的信息。
- 比 Relu、Gelu 效果更好。
- 在模型结构已经很深时，继续叠加层数不如使用 SwiGLU 加宽模型。
