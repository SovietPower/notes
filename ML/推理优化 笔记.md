## 基础

### 计算密集度

> https://docs.nvidia.com/deeplearning/performance/dl-performance-gpu-background/index.html#understand-perf

性能主要受三者影响：内存带宽 (memory bandwidth)，计算带宽 (math bandwidth)，延迟。

对于一个简单的函数（从内存读数据，计算，写数据到内存）来说，设 $T_{mem}$ 是访存的时间，$T_{math}$  是执行计算的时间，设不同线程的访存和计算可并行，那么该函数的实际耗时是 $\max\{T_{mem}, T_{math}\}$。如果前者更大，则函数是 memory limited；否则函数是 math limited。

$T_{math}$ 取决于计算量和处理器的数学带宽，$T_{mem}$ 取决于访存的字节数和处理器的内存带宽，因此 $T_{math}>T_{mem}$ 可以表示成：$\#ops/\#bytes > BW_{math}/BW_{mem}$。
$\#ops/\#bytes$ 即为计算密集度/计算强度 (*arithmetic intensity*, *computational intensity*)，$BW_{math}/BW_{mem}$ 即为操作数据比 (*ops:byte* ratio)。

> $\#ops$ 为 FLOPs (floating point operations)，即针对单个输入，模型完成一次完整的向前传播所发生的浮点运算个数。
> 计算 FLOPs 时有时会将 MAC (Multiply‒Accumulate) 视为一次浮点计算，尽管确实是乘+加两次。
> $\#bytes$ 为模型完成一次前向传播要发生的内存交换总量。
>
> $BW_{math}$ 也称作算力 (max performance, $P_{peak}$)，即计算平台每秒能完成的浮点运算数上限。
> $BW_{mem}$ 就是带宽 (bandwidth, $b_s$)，即计算平台每秒能完成的内存交换量上限，单位为 byte/s (Bps)。
>
> 计算强度上限 (max Computational intensity, $I_{max}$) 即为 $BW_{math}/BW_{mem}=P_{peak}/b_s\ FLOPs/Byte$。
> 模型的计算强度应尽可能接近算力上限 $I_{max}$，但超出之后就无意义了。

如果一个算法的计算密集度 > 操作数据比，则它是 math limited；否则是 memory limited。
（因为硬件带宽是固定的，所以具体怎样就看算法的计算密集度）

> 计算密集度越高，内存使用效率越高。
> 因为计算密集度的计算很简单，所以这只是一个非常理论的参考值：
>
> - $T_{math}$ 只考虑了运算指令、没考虑其它指令的执行时间。
> - $T_{mem}$ 假设输入输出只会各访存一次。
> - $\#ops,\#bytes$ 应该(?)是每秒/一段时间内的运算数/访存量？不是程序整体的，难以计算。
> - 实际带宽 $BW_{mem}$ 还取决于数据从哪里读取、cache 命中率。

常见的 op 可以有几类：

- Elementwise Operations：通常是对两个或一个 tensor 逐元素进行运算。通常会读取大量内存，但只做少量运算。
  包括大部分非线性运算 (relu, sigmoid...)、scale、add 等。
- Reduction Operations：在一组值上进行运算。同上通常计算密度很低。
  包括 pooling、batch normalization 计算 mean 和方差的部分、soft max。
- Dot-Product Operations：这类操作可以表示为两个 tensor 元素的点积（通常是权重与激活值（即输入））。
  包括全连接层、卷积（一组点积）等。
  如果操作矩阵足够大，那么它可能是 math bound 的；否则与 elementwise op 类似可能是 memory bound 的。

### 矩乘

> https://docs.nvidia.com/deeplearning/performance/dl-performance-matrix-multiplication/index.html

广义矩乘 (GEMM, General Matrix Multiplications) 指操作 $C=\alpha AB+\beta C$。

当有大量或很大的矩乘影响性能时，应尽量让它使用 tensor core 并避免 Quantization Effects。见 *cuda - tensor core*。

**计算密集度**

对于 $C=\alpha AB+\beta C$，设矩阵 A 为 M \* K，B 为 K \* N，C 为 M \* N，则 $A*B$ 包括 M \* N 个 K 个元素的点积。
计算 $A*B$ 需要 M \* N \* K 个 fused multiply-adds (FMAs)，每个 FMA 包括乘和加两个操作，所以总共需要 2 \* MNK FLOPS。

所以矩乘的计算密集度为（忽略系数的乘积）：

$$ops:byte=\frac{FLOPS}{bytes\ to\ access}=\frac{2\cdot MNK}{2\cdot(MK+NK+MN)}=\frac{MNK}{MK+NK+MN}$$

对于一个 `M*N*K = 8192*128*8192` 的 gemm，计算密集度为 124.1 FLOPS/B，低于 V100 的 138.9 FLOPS:B，所以它在 V100 上是 memory limited；对于 `8192*8192*8192`，计算密集度为 2730，就是 math limited（注意这只是理论值）。

当 M 或 N 为 1 时，就成为了 matrix-vector products (general matrix-vector product, GEMV)，此时计算密集度始终小于 1，所以为 memory limited。

> 这里访存量就是三个矩阵大小的和，假设了相同元素只访存一次，所以非常理论。
> 这种分析下矩阵尺寸的增加对计算量 MNK 的影响比访问量 MN 更大，所以访存密集 gemv 会变成计算密集 gemm。





### 全连接层

> https://docs.nvidia.com/deeplearning/performance/dl-performance-fully-connected/index.html

全连接层 (fully-connected layers) 也叫 *线性层* (linear layers)，指上一层的神经元（输入）和下一层的每个神经元（输出）都有连接。
影响其性能的因素主要为：batch size，输入神经元数量，输出神经元数量。

简单的[总结](https://docs.nvidia.com/deeplearning/performance/dl-performance-fully-connected/index.html#checklist)：

- Choose the batch size and the number of inputs and outputs to be divisible by 4 (TF32) / 8 (FP16) / 16 (INT8) to run efficiently on Tensor Cores. For best efficiency on A100, choose these parameters to be divisible by 32 (TF32) / 64 (FP16) / 128 (INT8) refer to Tensor Core Requirements.
  - 主要的 GEMM 都有 bs 参与，所以可以将 bs 调整为对应的倍数来满足这一点（输入输出可以无所谓）。
  - 这同样适用于训练，所以最好还是直接将矩阵大小设为对应倍数。
  
- Especially when one or more parameters are small, choosing the batch size and the number of inputs and outputs to be divisible by at least 64 and ideally 256 can streamline tiling and reduce overhead; refer to Dimension Quantization Effects.
  - 见 *cuda 笔记 - tile quantization, wave quantization*。

- Larger values for batch size and the number of inputs and outputs improve parallelization and efficiency; see Performance.
  - 全连接层就是 GEMM，而更大的 GEMM （即更多的输入输出、参数）通常效率更高（见 *cuda 笔记 - typical tile dimensions*）。
  - batch size 的大小直接影响前向传播和激活值梯度计算时的 GEMM 矩阵大小（就是 N 或 M）；在权重梯度计算时，bs 虽然不影响 GEMM 大小，但是会作为 GEMM 中的 K 影响每个 tile 的计算效率（越大效率越高）。因此（在一定范围内）更大的 bs 效率更高、更能充分利用 GPU。
  - 非常小的 bs 会使计算变成 memory bound。

- choose batch sizes and neuron counts greater than 128 to avoid being limited by memory bandwidth (NVIDIA® A100-SXM4-80GB; this threshold is similar for other A100 and V100 GPUs); see Batch Size.

### 算子

> 算子是 AI 框架中最基础的概念，它是对张量 (Tensor) 执行基本操作的集合，包括四则运算、数学函数、甚至是对张量元数据的修改，如维度压缩（Squeeze），维度修改（reshape）等。
> Kernel 是算子的底层实现，通常是用 C++ 等语言编写的。Kernel 通常不感知微分的概念，也不支持自动梯度计算。
> 中间态是指在计算过程中产生的临时数据，它可以是张量，也可以是其他类型的数据。
>
> 简单来说：
>
> - 算子：从数学角度定义的操作。
> - Kernel：算子的底层实现。
> - 中间态：计算过程中的临时数据。
>
> 深度学习模型由一个个计算单元组成，称这些计算单元为算子 (Operator，简称 OP)。在网络模型中，算子对应层中的计算逻辑，例如：卷积层 (Convolution Layer) 是一个算子；全连接层 (Fully-connected Layer, FC layer) 中的权值求和过程，是一个算子。
>
> 实现上，算子可以是一个类或函数（编程中的函数，不是数学意义的函数）。

> 函数是从数到数的映射，泛函是从函数到数的映射，算子是从函数到函数的映射。
> 有时这几个词可以混用，比如可以把数当作常函数，那么普通的函数也可以看作泛函或算子；再比如考虑从算子到算子的映射，你仍然可以叫它算子。
>
> 可以认为算子是把一个函数变成另一个函数的东西。
> 梯度（求导）就是一个算子，将一个函数映射到另一个函数。





### 张量

张量就是任意维度的数组，是对向量和矩阵的扩展。

张量的形状 (shape) 即每个维度的大小。比如：shape = (4, 20, 20, 3) 可以代表有4张图，每张图的宽和高都是20（400个像素），每个像素点都由 RGB 3色组成。

> 在深度学习领域，多维数据通过多维数组存储，比如：卷积神经网络的特征图 (Feature Map) 通常用四维数组保存，即 4D 格式：
>
> N：Batch数量，例如图像的数量。
> H：Height，特征图高度，即垂直高度方向的像素个数。
> W：Width，特征图宽度，即水平宽度方向的像素个数。
> C：Channels，特征图通道，例如彩色RGB图像的Channels为3。
>
> 由于数据只能线性存储，因此这四个维度有对应的顺序。不同深度学习框架会按照不同的顺序存储特征图数据，比如 Caffe 中排列顺序为 [Batch, Channels, Height, Width]，即 NCHW；TensorFlow 中，排列顺序为 [Batch, Height, Width, Channels]，即 NHWC。
>
> 以一张格式为 RGB 的图片为例，NCHW 中，C 排列在外层，实际存储的是“RRRRRRGGGGGGBBBBBB”，即同一通道的所有像素值顺序存储在一起；而 NHWC 中 C 排列在最内层，实际存储的则是“RGBRGBRGBRGBRGBRGB”，即不同通道的同一位置的像素值顺序存储在一起。

## 常见优化

> 模型压缩: 包括参数压缩和层剪枝、剪枝、量化等优化技术，通过降低模型复杂度来实现推理计算的加速。硬件加速: 利用特定硬件设备（如GPU、ASIC、FPGA等）相较于通用CPU有更高的并行计算能力，还可以针对特定算子进行优化，进而加速模型推理。图形优化: 该技术主要针对神经网络模型图结构进行优化，包括图分解等方法，同时还需要考虑到推理引擎的限制，以使其能够最大限度地利用硬件资源。运行时优化: 利用实际应用场景中大量存在的稀疏性、数据重用等特点，对模型计算过程中梯度更新、前向传递等操作进行优化，降低计算消耗。并发执行: 在支持并发运行的硬件上采用多线程或异步调度等技术，充分利用系统资源，提高模型推理的效率。预编译模型: 将模型完全编译后，可以避免模型的重复解析和优化计算，提高了模型推理速度和稳定性。

> 部分 layer 的简单优化：https://docs.nvidia.com/deeplearning/tensorrt/archives/tensorrt-803/best-practices/index.html#optimize-layer



### Batching

通过 batching 尽可能提高计算并行度是很重要的优化。
以下来自 TRT 文档：

- 推理时网络中的每一层都会有同步开销。一次并行计算更多的结果，可以降低它的影响。
- 许多层的性能会被输入中最小的维度所限制。例如：对输入数量为 V（输入特征向量的维度）、输出数量为 K 的全连接层（即权重矩阵大小为 V\*K），N 次 `1*V` 和 `V*K` 矩阵的向量乘（bs = 1）要远比一次 `N*V` 和 `V*K` 的矩阵乘（bs = N）要慢。

在 GPU 上使用更大的 bs 几乎总是更高效的。但过大的 bs（如 > $2^{16}$）可能需要额外的索引计算，所以需要避免。
如果网络有矩乘或全连接层，建议 bs 使用 32 的倍数来充分利用 tensor core。



### cuda graph

有时推理性能会被 kernel launch time（如 TRT execution context 的 enqueue/enqueueV2 耗时）而非 GPU 计算所限制。例如：bs 较小或网络中存在许多 kernel 执行时间较短的 layer 时，kernel launch time 会占据推理耗时的主导。
使用 trtexec 对比 Enqueue Time 和 GPU Compute Time，或用 nsys 对比 kernel launch 和 execution 时间，可以检查是否有该问题。

cuda graph 可以降低 launch 耗时。




### 混合精度训练

> https://zhuanlan.zhihu.com/p/535558158
> https://zhuanlan.zhihu.com/p/103685761
>
> https://www.tensorflow.org/guide/mixed_precision?hl=zh-cn#xla
> https://www.intel.com/content/www/us/en/developer/articles/guide/getting-started-with-automixedprecisionmkl.html

混合精度训练使用多种精度进行训练。使用低精度值训练有以下特点：

- 提升计算效率：现代 GPU 特别是特殊的显卡为 FP16 进行了专门优化，使用 FP16 执行计算能够比 FP32 更快。
- 降低内存消耗：FP16 占用的内存是 FP32 的一半，能够减少显存消耗，训练更大的模型和使用更多的数据。
- 提高通讯性能：减少分布式训练中网络通信的影响。

低精度值也会带来一些问题：

- 数据溢出：FP16 的有效范围要小很多，可能出现上溢和下溢。在 DL 中需要计算网络模型中权重的梯度（一阶导数），因此梯度会比权重值更小，容易出现下溢。
- 舍入误差：FP16 的精度更低，无法表示 FP32 能正常表示的值（[这里](https://en.wikipedia.org/wiki/Half-precision_floating-point_format#Precision_limitations)展示了不同范围内的最小间隔）。

混合精度训练不完全依赖 FP16，以在利用 FP16 优势的同时避免其它问题。主要包含三点：

1. 权重备份 (Weight Backup)：把神经网络训练过程中产生的激活值 activations、梯度 gradients、中间变量等数据，在训练中都用 FP16 来存储，同时复制一份 FP32 的权重参数 weights，用于训练时候的更新，以减少舍入误差。
	模型更新即$weight = weight + \eta * gradient$，weight 和该运算使用 FP32 来避免 FP16 误差（但容易出现误差的不是梯度吗？）。
	会占用一部分内存，但算上 FP16 保存的动态内存，总体还是会减少。
2. 损失缩放 (Loss Scaling)：当梯度值太小时，FP16 可能导致下溢使模型无法收敛。损失缩放把前向计算出的损失值乘一个系数 (2^k)，使误差值移动到 FP16 可表示的范围。
	反向传播时，使用 FP32 计算并将梯度缩小 2^k 倍。
	根据链式求导，直接对 loss 应用缩放与在每一层进行缩放效果是一样的。
3. 精度累加 (Precision Accumulated)：使用 FP16 进行矩乘运算，使用 FP32 进行矩乘中间的累加 (accumulated) 弥补丢失的精度，然后再将 FP32 的值转化为 FP16 存储。这样可以有效减少计算过程中的舍入误差，减缓精度损失。

动态损失缩放 (Dynamic Loss Scaling)：使用可变的放大倍数，以更好地缓解舍入误差：每当梯度溢出时减少 loss 缩放比例，并且间歇性地尝试增加缩放比例，从而在不导致溢出的情况下使用最高缩放比例，更好地恢复精度。







### 算子融合

> https://tensorflow.google.cn/xla?hl=zh-cn



通过识别特定常见的 pattern，可以将多个 layer 融合成一个 layer，减少 kernel 调用和显存读写。

以层归一化 (Layer Normalization) 为例，tf 的实现为：

```python
mean = tf.reduce_mean(x, axis=[-1], keepdims=True)
variance = tf.reduce_mean(tf.square(x - mean), axis=[-1], keepdims=True)
result = (x - mean) * tf.rsqrt(variance + epsilon) * scale + bias
```

这存在3次 kernel 调用和2次显存读写。可以把三步操作融合为一步，用一个核函数，将 mean、variance 结果暂存入寄存器，减少显存访问。

> [常见的 fusion](https://docs.nvidia.com/deeplearning/tensorrt/archives/tensorrt-803/best-practices/index.html#fusion-types) 有：ReLU ReLU Activation（合并连续的两个 relu）、Convolution and ReLU Activation（合并卷积 + relu）等。移除无用的 layer（如 add 0 或 multi 1 的 Scale）也能算融合。



### beam search 优化

为了挑选概率 top k 的 token，需要在 [batch_size, beam_size, vocab_size] 大小的 logit 矩阵上进行 softmax 计算及显存读写，然后进行 batch_size 次排序取 k 大（每次排序规模为 beam_size * v_size？）。
通常 vocab_size 都是几万规模，因此计算量非常庞大。

LightSeq 借鉴搜索推荐中常用的粗选-精排两段式策略，将解码计算改成层级式：设计了一个 logit 粗选核函数，避免了 softmax 计算和对十几万元素的排序。该粗选核函数遍历 logit 矩阵两次：

- 第一次遍历：对每个 beam，将其 logit 值随机分成 k 组，每组求最大值，然后对这 k 个最大值求一个最小值，作为一个近似的 top-k 值（小于等于真实 top-k 值），记为 R-top-k。
- 第二次遍历：对每个 beam，找出所有大于等于 R-top-k 的 logit 值，进行精排（正常的排序逻辑）。可以保证结果正确，但大大降低了排序复杂度。

## 量化

> https://blog.csdn.net/m0_70420861/article/details/135552977



**线性量化**

根据量化数据表示的原始数据范围是否均匀，可以将量化方法分为线性量化 (Uniform Quantization) 和非线性量化 (Non-Uniform Quantization)。

线性量化最常用，根据量化后的值域是否围绕零点对称可分为两种：

- 对称量化：最常见的为绝对最大值 (absmax，absolute maximum) 量化：$x_{quantized}=\text{round}(s*x),\ s=\frac{127}{\max_i(\text{abs}(x_i)}$（8b 量化）。
- 非对称量化：$x_{quantized}=\text{round}(s*x+z), \ z=\text{round}(-s*\beta)-128,\ s=\frac{255}{\alpha-\beta}$。$\alpha,\beta$分别是最小最大值。
- 对称量化可以避免量化算子在推理中计算 zero-point 相关的部分，降低推理的计算复杂度；非对称量化可以根据实际数据的分布确定最大值和最小值，可以更加充分的利用量化数据信息，使量化导致的损失更低。







### 量化分类

> 低比特量化：传统的量化方法分为Quantization-Aware Training (QAT) 和 Post-Training Quantization (PTQ)，可以简单通过是否需要重新训练来进行区分.
> 模型量化（Model Quantization）：模型量化是一类应用广泛的模型压缩技术，其通过将模型的权重从高比特数转换成低比特数来降低模型的计算和存储开销。值得注意的是，大模型在推理的不同阶段有不一样的效率瓶颈，因此需要适配不同的量化方法。在预填充阶段，推理效率主要受限于较高的计算量，因此通常采用权重激活量化（Weight-Activation Quantization）的方式；在解码阶段，推理效率主要受限于较高的访存量，因此通常采用仅权重量化（Weight-only Quantization）的方式。从量化流程来看，模型量化可以划分为：
> 训练后量化（Post-Training Quantization，PTQ）：指对预训练后的模型直接做量化，不需要重新训练量化后的模型。这类方法对量化后模型的精度损失较大，但因其不需要额外的模型训练开销，因此在大语言模型领域应用最为广泛。训练感知量化（Quantization-Aware Training，QAT）：指在模型的训练中加入模型量化过程，并通过训练减小量化带来的精度损失。相比于训练后量化，训练感知量化方法通常具有更高的精度，但其需要大量的数据和计算量来做模型训练。

#### 量化感知训练 (QAT, Quantization Aware Training)

在训练过程中模拟量化：在两个层之间插入 fake quants，即将输出进行量化+反量化后再作为下一层的输入。
利用伪量化算子将量化带来的精度损失计入训练误差，使得优化器在训练中进行 loss 计算和权重更新时能够考虑到量化误差，从而尽量减少量化误差，得到更高的模型精度。

QAT 会尝试探索损失函数中的“宽”最小值区域，以尽可能减少量化误差：宽最小值区域内参数的变化对损失函数的影响较小，即使量化引入了一些小误差，模型性能也不会受到太大影响，这种鲁棒性使模型在量化后仍然能保持较好的性能；窄最小值区域内参数的变化对损失函数的影响较大，即使是微小的量化误差也可能导致较大的性能下降，且量化误差的累积效应也更大（即使每个参数的量化误差较小，这些误差在模型输出中也会被放大）。

如果不进行 QAT、在训练时不考虑量化误差，模型就会选择 loss 最小的权重，即使它位于窄最小值区域、会引入更大的量化误差。



#### 训练后量化 (PTQ, Post Training Quantization)

在 LLM 训练完成后再对其参数进行量化。需要少量校准数据，适用于追求高易用性和缺乏训练资源的场景。
优点是可以减少 LLM 的存储和计算复杂性，无需对 LLM 架构进行修改或重新训练，简单高效。
但 PTQ 可能会在量化过程中引入一定的精度损失。

PTQ 可分为两种：

- 权重量化：仅量化模型的权重以压缩模型的大小，在推理时将权重反量化为原始的 fp32，后续推理流程与普通的 fp32 模型一致。权重量化的优点是不需要校准数据集，不需要实现量化算子，且精度误差较小。
- 全量化：不仅量化模型的权重，还量化模型的激活值，在模型推理时执行量化算子来加快模型的推理速度。
  为了量化激活值，需要一定数量的校准数据集用于统计每一层激活值的分布，并对量化后的算子做校准。校准数据集可以来自训练数据集或者真实场景的输入数据，需要数量通常非常小。



#### 量化粒度

根据量化粒度即量化参数 s 和 z 的共享范围，量化可以分为：

- 逐层量化 (per-tensor)：以一层网络为量化单位，每层网络有一组量化参数。
- 逐通道量化 (per-token + per-channel, vector-wise quantization)：以每层网络的每个或多个向量为单位，分别有一组量化参数。
  - 对于 $X*W$，可以视作矩阵 X 的每一行和 W 的每一列独立计算内积，因此可以为 X 的每行 (per-token) 和 W 的每列 (per-channel) 都分别设置一组量化参数。
  - 引入更多量化参数，能使量化结果更准确，并将异常值的影响限制在每个组中，但会影响性能。
- 逐组量化 (per-group, Group-wise)：就是允许逐通道量化将多行和多列分为一组，分别进行量化和内积，是前两者的折中。



#### LLM.int8(), 混合精度分解 (Mixed-precision Decomposition)

类似逐组量化，可以分解激活值（按行）和权重（按列）矩阵，对绝大部分激活和权重用 8bit 量化（计算后再转为 FP16），对包含异常值的行/列保留原始精度进行高精度矩阵乘法，最后用 FP16 相加得到高精度结果。

可以在降低显存占用的同时保持模型精度，但会导致推理变慢。

### 权重剪枝

在大多数神经网络中，权重的数值分布接近正太分布（或多正太分布的混合），且越接近于 0 权重越多，这就是权重稀疏现象。
权重数值的绝对值大小可以看做重要性的一种度量，权重数值越大对模型输出贡献也越大，反正则不重要，删去后模型精度的影响应该也比较小。
*权重剪枝* (Weight Pruning) 就是删除数值较小的权重。

不同类型、不同顺序的网络层，在权重剪枝后影响各不相同。

- 卷积层的剪枝敏感性大于全连接层，且第一层卷积层最为敏感。
  可能是全连接层本身参数冗余性更大；第一层卷积层冗余性更少。

即使是移除绝对值接近于 0 的权重也会带来推理精度的损失。为了恢复模型精度，通常在剪枝后需要再训练模型。
常规的模型剪枝 pipeline 有三步 (three-stage training pipeline)：

1. 训练。
2. 剪枝，将低于阈值的权重删除。
3. 重新训练/微调，然后重复 2~3，直到模型规模和精度达标。

权重剪枝是非结构化稀疏 (unstructured sparsity) / 细粒度稀疏（等价于在矩阵中随机移除某些元素/某行/某列），非常不规则，对 GPU 等并行计算不友好甚至没有效果。

**如何评估连接权重的重要性**

1. 基于神经元连接权重数值幅度，就是权重绝对值是否超过某阈值。
2. 使用目标函数对参数求二阶导数表示参数的贡献度。

### 激活稀疏

以 ReLU(x) = max(0, x) 为例，激活函数通常产生大量 0 输出，给网络带了另一种类型的稀疏性。
此外 max_pooling 只会保留网络中能成为最大值的值，即会将大量神经元的输出视为 0。

因为神经网络是乘法-加法-激活的计算过程，因此输出大部分为零的神经元对后续层的输出以及最终结果的贡献很小，即零神经元很可能是冗余的，可以在不影响网络整体精度的情况下将其移除，它们的存在只会增加过度拟合的机会和优化难度。

用 APoZ (Average Percentage of Zeros) 表示某个神经元经过 ReLU 映射后零激活值的百分比（输入多个样本，结果为 0 的样本数占总样本数的百分比），训练过程为：

1. 在大规模数据集上运行网络获得每个神经元的 APoZ。
2. 按特定标准移除高 APoZ 的神经元，移除与其相关的权重和计算。
3. 重新训练网络以加强剩余的神经元。

*激活剪枝*剪枝的对象是神经元，即将某些神经元的输出置零，属于结构化稀疏 / 粗粒度稀疏，不是靠影响矩阵本身的稀疏程度，因此适用于 GPU 等。



### 梯度稀疏

大多数模型的梯度是稀疏的。随着训练周期的增加，参数梯度的稀疏度可能会显著增大。

梯度会导致网络带宽成为分布式训练的瓶颈。为了减少通信，可以根据梯度稀疏进行剪枝：

1. 仅传输一定比例的梯度。
2. 仅传输绝对值超过阈值的梯度。
3. 深度梯度压缩



### 其它结构化剪枝

- 非结构化剪枝 (Unstructured Pruning) 对模型的单个参数/权重进行剪枝，具有更高的模型压缩率和准确性，但在通用硬件上的加速效果不好。因为其计算特征上的“不规则”，需要特定硬件才能实现加速效果。
- 结构化剪枝 (Structured Pruning) 虽然牺牲了模型压缩率和准确率，但在通用硬件上的加速效果好，所以被广泛应用。结构化稀疏使得权值矩阵更规则更加结构化，更利于硬件加速。

- ClearList (numerically-neural)：安全，但不太影响性能，视情况决定是否转 f16。

## 其它优化

### 3xTF32

> https://github.com/NVIDIA/cutlass/discussions/385

**TF32**

TF32 (TensorFloat-32) 的符号/指数/尾数为 1/8/10，虽然占用 32b 内存，但实际计算时仅使用 19 位。
与 FP32 相比，TF32 在 Ampere GPU 上通过 Tensor Core 运行，计算吞吐量是 FP32 的 8-16 倍（与 FP16 一致），同时精度接近 FP32（优于 FP16）。

> FP32 为 1/8/23（指数位相同），FP16 为 1/5/10（尾数位相同，所以 TF32 精度与 FP16 一致）。

**3xTF32**

3xTF32 指将 FP32 的矩乘替换为 3 个 TF32 矩乘，后者可以模拟前者的精度，同时性能提升两倍。

原理为：

```
a x b = (a_big + a_small) x (b_big + b_small) = a_big x b_big + a_big x b_small + a_small x b_big
big = fp32_to_tf32(fp32)
small = fp32_to_tf32(fp32 - big)
```

FP32 本就不可表示 a_small * b_small 的结果，所以可忽略（不忽略结果也会不对）。
