**快排**
平均复杂度（$T(n)=T(i)+T(n-i)+O(n)$）证明可以枚举$i\in[1,n]$然后加起来取平均（比较麻烦），也可以计算两个元素的期望被比较次数。

```
void qsort(int l,int r)
{
	int i=l, j=r;
	int x=a[Rand%(r-l+1)+l];
	while(i<=j)
	{
		while(i<=j && a[j]>x) j--;
		while(i<=j && a[i]<x) i++;
		if(i<=j)
		{
			std::swap(a[i],a[j]);
			i++, j--;
		}
	}
	if(l<j) qsort(l, j);
	if(i<r) qsort(i, r);
}

template <class FwdIt, class Compare = std::less<>>
void quickSort(FwdIt first, FwdIt last, Compare cmp = Compare{})
{
    auto const N = std::distance(first, last);
    if (N <= 1) return;
    auto const pivot = std::next(first, N / 2);
    std::nth_element(first, pivot, last, cmp);
    quickSort(first, pivot, cmp);
    quickSort(pivot, last, cmp);
}
```

**拓扑排序**

对于一个 DAG，生成一个原来节点的排列，排列满足：如果 x 出现在 y 的前面，则 x 在 y 的下游，即有可能存在 y 到 x 的路径，但一定不存在一条 x 到 y 的路径。
可以解决一些依赖关系问题，比如 A 依赖 B,C，则 B,C 出现在 A 的前面，能保证只有 B,C 完成，才会选 A。

**堆**
push 时放到最后面，然后不断向上直到和父节点的大小关系满足。
pop 时将最后一个元素放到 top 位置，然后放到下面。
相比快排，堆排的常数更大。

```cpp
struct Heap // 小根堆
{
	int sz,A[M];
	inline int Top() { return A[1]; }
	void Push(int x)
	{
		A[++sz]=x;
		int now=sz, nxt=now>>1;
		while(now>1 && A[nxt]>A[now])
			std::swap(A[nxt],A[now]), now=nxt, nxt=now>>1;
	}
	void Pop()
	{
		A[1]=A[sz--];
		int now=1;
		while(now<<1<=sz)
		{
			int nxt = now<<1;
			if(nxt<sz && A[nxt]>A[nxt+1]) ++nxt; // 如果不用nxt+1，与nxt交换后会导致父节点>子节点
			if(A[nxt]>=A[now]) break;
			std::swap(A[now],A[nxt]);
			now=nxt;
		}
	}
};
std::priority_queue<pr, std::vector<pr>,std::greater<pr> >; // 小根堆，对结构体可以重载运算符（默认是大根堆，重载 < 时按 > 来为小根堆）
```

介绍和复杂度证明：
堆是一个二叉树，以小根堆为例，保证父节点的值一定小于等于子节点。所以树根的值一定是最小值。
取出树根后，我们将一个叶子放到根，然后根据与子节点的大小关系将它再下放到叶子节点。
插入数据时，我们将它放到叶子节点，然后根据与父节点的大小关系将它一步步上浮到根。
可见复杂度取决于树的高度。如果我们用完全二叉树（任意两个叶子节点的深度不会超过1）的形式组织树，高度就是 log n 了。
具体步骤看一下代码就很清楚。

**基数排序**

基数排序是基于内容而非基于比较的排序。可用于数字、字符串这种，能分为若干位，且每位值域很小的情况。

过程：在最低位上进行桶排：将所有元素放入指定桶后，依次取出得到新序列。然后将新序列在次低位上进行桶排，...
保证了 进入同一个桶的数据 在更低位的部分是有序的：在排序第i位时，数列是按照1...i-1位有序的，当根据第i位放入桶后，它们在桶内也是按照1...i-1位有序的，再全取出来就是按照1...i位有序的。

设基数为 b，值域为 M，则复杂度为 $O((\lfloor\log_bM\rfloor+1)\cdot(n+b))$，其中 $d=\lfloor\log_bM\rfloor+1$ 为迭代次数。
如果基数取10或2，则 d 为 O(logM)，复杂度为 O(nlogM)，效率一般。
越大的base需要的桶越多，但效率越高（需要的轮数少）。
所以当值域有限时（如 M<=n^c），取 base=n 即为 O(n)，但需要 n 个桶。

> 1. 基于比较的排序，有个隐含的假定：关键字比较的开销为O(1)。
> 2. 对长度为n的数组排序，如果数组的 key 都不相同，那么根据信息论，每个关键字的长度是log2(n)：
>     决策树是一个二叉树。每进行一次比较，我们就能在决策树上走进下一层。所以决策树的高度就是我们比较的次数。
>     决策树至少（最优放置情况下）有 n! 个叶子，故高度至少为 $\log(n!)\geq n\log n$。
>     所以基于比较的算法，nlogn 就是最优解了。
>
> 更多的是内存上的考虑。作为一种通用的排序方法，最好不要带来意料之外的内存开销，所以各语言的默认实现都没有用基数排序，但是不能否认其在各领域的应用。

**红黑树**

特点：每个节点有一个颜色，红色或黑色；根是黑色的；红色节点不能相邻（父子节点不能都为红色）；从根开始到 NIL 的任意一条路径上，黑色节点数相同（每个节点都有左右儿子，如果不指向实际节点则为 NIL）。

各操作复杂度与树高度相关，需要很多分类讨论。

树高度是$O(\log n)$的。
证明：设k为最大的满足所有 root-NIL 路径都至少有k个节点的k值，则显然树包含一棵高恰好为k的完美二叉树，包含2^k-1个节点。
所以有$n\geq 2^k-1$，得$k\leq \log(n+1)$。
由假设，存在一条root-NIL路径，上面只有k个节点。在树最高，也就是黑节点尽量多的情况下，这条路径全是黑节点，所以每条 root-NIL 上黑节点的个数为k。
对每条 root-NIL 路径，因为红节点只能出现在两个黑节点之间（或叶子），所以红节点最多有k个，所以路径长度最多为2k。即树高最大为$2k\leq 2\log(n+1)$。

**AVL 树**

> https://blog.csdn.net/xiaojin21cen/article/details/97602146

特点：记一个节点的高度为：$h[x]=\max\{h[lson_x], h[rson_x]\}+1$，即到子树中某个叶子节点的最远距离。
则对任意一个节点，其左右儿子的高度差不超过1。

维护过程：假设插入 a 后，节点 x 的左右儿子高度差超过了1。
如果 a 是在 x 的左儿子的右子树中插入的，则进行左右旋（先在 x 的左儿子处左旋，再在 x 处右旋）。
如果 a 是在 x 的左儿子的左子树中插入的，则进行左左旋。
如果 a 是在 x 的右儿子的左子树中插入的，则进行右左旋。
如果 a 是在 x 的右儿子的右子树中插入的，则进行右右旋。

**B+ 树**

设每个节点最多有 n 个指针，则对树上的每个非叶节点，必须有$\lceil\frac n2\rceil\sim n$个儿子；对每个叶节点，必须有$\lceil\frac{n-1}{2}\rceil\sim n-1$个值（指针有 n 个，值最多有 n-1 个）。

- 搜索：与其它平衡树一样，根据大小关系决定走哪个指针，直到走到叶子。
- 插入：首先找到数据应在的叶子，如果叶子没满就插入，否则将叶子拆分成两个，各包含一半的值。拆分叶节点后，将新的叶子插入到它父节点的儿子中，如果父节点的指针没满就插入，否则将父节点也拆分成两个。这样递归向上就可以了。
    最坏情况下，从叶子到根节点的路径上的所有点都要被拆分。
    如果根节点被拆分，则需要一个新的根节点，整棵树的深度 +1。
- 删除：首先找到数据应在的叶子，删除。如果删除导致节点中的数据数量小于下限，检查它的左兄弟节点，如果两个节点可以合并，数量不超过上限，就合并；否则将兄弟节点的最右边的数据移动给它。
    如果叶节点被合并，导致父节点中的数据数量小于下限，执行同样的过程。递归向上。

**红黑树 与 AVL 树 的比较**

> 总结：红黑树与 AVL 树没有特别显著的差距，STL 为什么用红黑树也没有明确的说法，可能是因为多数场景下，更新操作比较多，红黑树更好；如果查询非常多，AVL 就要好一些。

AVL 树比红黑树更平衡，所以高度更小，查找要稍微快点（AVL 的最大高度为 1.44logn，红黑树是 2logn）。

但 AVL 树的更新慢一点：为了更平衡，一次更新需要的操作就更多，可能要在 目标节点到根的所有节点处 都进行一次旋转。
而红黑树的一次更新，最多只会旋转 3 或 3 次（不过可能染色很多次）。旋转是多个指针操作，所以 AVL 的大量旋转可能会影响性能。

此外，红黑树的操作可以非递归实现，也许会有一点点影响，但层数太少了。而且好像会影响其它操作复杂度？

**红黑树 与 B+ 树 的比较**

> 总结：B+ 树的连续性更好，可以利用内存读取的单位为 cacheline 这一特点（很多服务器上的 cacheline 可能能到 512B），性能理论上可能更好。
> 此外除了 B+ 树指针失效的问题外，和红黑树也没有太明显的差距，也是可选择的，不过可能比较麻烦。
>
> 理论上 BST 的效率要优于 B+ 树，但这是基于内存均质随机访问的前提，现实中因为各级 cache 的存在 B+ 树效率反而可能更好。

B+ 树的更新，会在节点间移动数据，导致指针或迭代器失效，所以 C++ 的标准库肯定不会用（为什么失效不知道）。

B+ 树是一个多叉树，每个节点可以存放多个数据，用于磁盘可以减少 IO 次数，用于内存也更缓存友好，更适合连续访问，效率其实也很高。
但是同 AVL，B+ 树的一次更新，可能导致叶子到根的所有节点，发生拆分或合并，这包含很多指针操作。
而红黑树的一次更新，最多只会导致 3 次旋转，涉及的指针操作很少。


**红黑树 与 跳表 的比较**

- 跳表能做的，平衡树也都能做（比如范围查询/节点前驱后继）。
- 跳表是期望$O(\log n)$，所以复杂度不会更好。
- 随机的跳表实现简单，但更优的跳表也很难实现，所以实现上也不会很有优势。
- 特别：跳表操作涉及的节点少，如果是并发场景，跳表需要加锁的粒度更小（或易于实现无锁）（但 redis 是单线程的）。

综上，红黑树基本各方面都更优。

redis 使用跳表的原因有三个：

- 内存占用灵活：通过降低概率 p、减少层数，可减少内存占用（以牺牲性能为代价）。
- 易于实现范围查询（因为有 ZRANGE 等需求），且缓存局部性不比平衡树差。
- 便于实现、调试和扩展。

跳表各个操作的效率都比不过 B+ 树（见[这里](https://www.zhihu.com/question/595685448/answer/3019350614)），但 redis 的效率瓶颈不在这里（在网络 IO）。

**跳表**

基于随机化的算法。
能实现有序插入、删除、查找某个元素（哈希表），随机访问第k个元素（链表维护距离下一个节点实际隔了多少个节点），各操作复杂度为期望$O(\log n)$；遍历。

将有序链表分为多层，最底层是原链表，越高层的节点数越少，每走一步跨越的节点就越多。
第 i 层的每个节点，有 p 的概率仍在 i+1 层出现。

查询 x 时，从最高层开始，如果当前节点的下一个节点 key 大于 x，则进入下一层；直到到达最后一层。

插入节点时，首先随机获得它要插入到哪一层：

```cpp
int RandomLevel() {
    int level = 1;
    while ((random()&0xFFFF) < (P * 0xFFFF))
        level ++;
    return (level < MAXLEVEL) ? level : MAXLEVEL;
}
```

即每次有 p 的概率层数+1，有 1-p 的概率结束。
所以每个节点的期望最高层数为$\sum_{i}i\cdot(1-p)\cdot p^{i-1}=\frac{1}{1-p}$（或$E=(1-p)E+p(E+1)$），所以期望节点总数为$\frac{n}{1-p}$。
因为 p 是常数，所以空间复杂度为期望$O(n)$。
最大层数通常为 log n（如果很高则效率低，如果很低则不能保证各层按照 p 分布）。所以最差情况下有 log n 层，最差空间复杂度为$O(\log n)$。

> 简单证明：一个节点能出现在第 log n 层的概率为$p^{\log n}=n^{\log p}$，取$p = \frac12$，则概率为$n^{-1\cdot\log_22}=\frac 1n$，所以第 log n 层节点个数的期望为$n\cdot\frac 1n=1$个，所以期望层数为 log n。
> p 还可以取得更小。
>
> 证明$a^{\log b}=b^{\log a}$：两边同时取对数，都等于：$\log a\cdot \log b$。

在插入 x 前，先判断有没有；然后计算要插入的层数 d，通过和查询一样的步骤，将 d 层及以下的最后一个小于 x 的节点后插入一个 x 的新节点（先通过查询标记出这些点）。
删除 x 类似，从最底层开始，通过和查询一样的步骤，检查最后一个小于 x 的节点后有没有 x，有就删除，没有就结束。
实现可参考[这里](http://oi-wiki.com/ds/skiplist/)。

**LRU-K**

只有访问至少 k 次的数据，才能进入缓存，否则留在另一个队列中。缓存中的数据与 LRU 一致。
LRU 的思想是：一个被使用的数据在不久后很可能被再次使用。但有些数据可能只使用一次，直接进入缓存会污染缓存。
LRU-K 可部分避免这个问题，实现更高的命中率，但维护代价也更高。
一般 LRU-2 最合适。MySQL 的新生代和老年代也可看做 LRU-2。

除了缓存的 LRU 链表外，还需要一个链表维护不在缓存的对象。每次访问取出给次数+1即可。
不在缓存的对象，也可通过 LRU 或 FIFO 等策略驱逐。

**枚举子集**

```cpp
for(s=0; s<all; ++s)
    for(i=s; i; i=(i-1)&s)
        ...
```

复杂度证明：设集合有$n$个元素，我们把所有子集$s$按元素个数$k=|s|$分类（因为它们的子集都为$2^k$个）。那么枚举的集合数为：
$$\sum_{k=0}^nC_{n}^k2^k=(1+2)^n=3^n$$

**枚举子集2**
枚举 [0, 2^n) 里恰好 k 个 1 的数字，要O(C(n, k))。

```c
void subset(int k, int n, T&& f) {
    int t = (1 << k) - 1;
    while (t < 1 << n) {
        f(t);
        int x = t & -t, y = t + x;
        t = ((t & ~y) / x >> 1) | y;
    }
}
```

**侵入式/非侵入式链表**

> https://zhuanlan.zhihu.com/p/679066486
>
> 把链表的结构（主要是pre next）放到数据里面去，这样数据就可以存在在多个链表中（存不同next即可）。由于此时链表结构在数据内部，而不是与数据一起在链表内部，因此需要额外自行管理生命周期，不是由链表控制。
> 优点是内存连续，取数据不需要通过指针间接寻址，且只需要一次分配内存。
>
> 侵入性：假设有两个类 A, B，它们要协同工作，如果 A 需要 B 提供特殊的设计才能一起工作，则称 A 有侵入性/ A 侵入了 B。
> 侵入式链表就是指：链表数据结构需要被存储的数据提供一些能力或代码修改，才能使用这个链表。
>
> 侵入式是一种设计模式或特点，有时会很方便。
> 类似的还有 *侵入式多态*。

**打乱一个序列**

Fisher-Yates 洗牌算法：从数组最后一个元素开始，将其随机的和它前面的某个元素交换；然后将倒数第二个元素和它前面的某个元素交换；将倒数第三个元素和它前面的某个元素交换...
也可以正序，但倒序很方便。

洗牌算法也称为等概率洗牌算法，对每个数来说，被交换到某个位置（在某个位置出现）的概率都相等，为1/n。
一个数被交换到最后概率均为1/n，被交换到n-1位置的概率为$\frac{n-1}{n}*\frac{1}{n-1}=1/n$，被交换到n-2位置的概率为$\frac{n-1}{n}*\frac{n-2}{n-1}*\frac{1}{n-2}=1/n$ ...

```go
// Fisher-Yates
for i := len(index)-1; i > 0; i-- {
    randId := rand.Intn(i+1)
    index[i], index[randId] = index[randId], index[i]
}
// std
rand.Shuffle(len(index), func(i, j int) {
    index[i], index[j] = index[j], index[i]
})
```

**从 n 个数中随机取 m 个数**

最简单的就是 random_shuffle 这个排列，取前 m 个。
另一个就是，首先随机取一个，然后再在 n-1 个数中随机取另一个... 重复。
这样每个数被取到的概率也都是 1/n：第一个显然，第二个$\frac{n-1}{n}*\frac{1}{n-1}=1/n$，第三个为$\frac{n-1}{n}*\frac{n-2}{n-1}*\frac{1}{n-2}=1/n$ ...
实现也类似洗牌，每次把随机到的数 交换到序列尾部，然后 n-- 继续随机、交换。

```cpp
template <typename It>
auto slide(It first, It last, It pos) -> std::pair<It, It>
{
    if (pos < first) return { pos, std::rotate(pos, frist, last) };
    if (last < pos) return { std::rotate(first, last, pos), pos };
    return { first, llast };
}
```

**gather**

```cpp
template <typename BiIt, typename UnPred>
auto gather(BiIt f, BiIt l, BiIt p, UnPred s) -> std::pair <BiIt, BiIt>
{
    return { stable_partition(f, p, not1(s)),
             stable_partition(p, l, s) };
}
```
